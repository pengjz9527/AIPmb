# 客户身份识别与KYC流程

## 流程概述

客户身份识别（Know Your Customer, KYC）是银行在建立业务关系时对客户提供的基本身份信息和其他必要信息进行核实和了解的过程。KYC是反洗钱和反恐怖融资的第一道防线，也是银行风险管理的基础。

根据《金融机构客户身份识别和客户身份资料及交易记录保存管理办法》（中国人民银行令〔2007〕第2号）及后续修订，银行须在与客户建立业务关系时进行身份识别，并在存续期间持续更新和重新识别。

KYC分为三个阶段：初次识别（建立关系时）、持续识别（存续期间）和重新识别（发现异常时）。

## 前置条件

### 适用业务范围

| 业务场景 | KYC要求 |
|----------|---------|
| 开立银行账户 | 初次识别（完整KYC） |
| 一次性大额交易（5万元以上） | 简化识别 |
| 贷款申请 | 初次识别+增强识别 |
| 理财产品购买 | 初次识别+适当性评估 |
| 跨境汇款 | 初次识别+资金来源核实 |
| 代理业务 | 双方身份识别 |
| 高净值客户（私行客户） | 增强型尽职调查（EDD） |

### 所需材料

| 客户类型 | 基本材料 | 辅助材料 |
|----------|----------|----------|
| 中国公民 | 居民身份证 | 户口簿、驾驶证、社保卡 |
| 港澳台居民 | 港澳台居民来往内地通行证 | 居住证 |
| 外国人 | 护照 | 居留许可、工作许可证 |
| 未成年人 | 户口簿/出生证明 | 监护人身份证 |

## 操作步骤

1. **客户身份初次识别** → 2. **身份信息核验** → 3. **风险等级评定** → 4. **持续身份识别** → 5. **异常触发重新识别** → 6. **尽职调查报告** → 7. **档案保存与更新**

## 各环节详解

### 一、初次识别

#### 识别内容

1. **基本信息采集**：
   - 姓名、性别、国籍、职业
   - 身份证件类型、号码、有效期
   - 联系方式（手机号、地址）
   - 常住地址（与证件地址不同时）
   - 开户目的和预期交易特征

2. **身份核验**：
   - 身份证阅读器读取证件信息
   - 联网核查系统验证身份证真实性
   - 人脸识别与证件照片比对
   - 公安部人口信息库比对

3. **实际控制人识别**：
   - 确认账户的实际使用人和受益人
   - 对于代理开户，核实代理关系
   - 确认资金来源的合法性

4. **政治公众人物（PEP）筛查**：
   - 核查客户是否为国内外政治公众人物
   - 核查客户亲属和密切关系人
   - 对PEP客户实施增强型尽职调查

#### 联网核查操作

1. 将身份证放入阅读器读取信息
2. 系统自动向人民银行联网核查系统发送请求
3. 核查系统返回比对结果：
   - **一致**：姓名、证件号、照片均一致
   - **不一致**：信息不匹配，需进一步核实
   - **无记录**：系统中无此证件信息
4. 核查结果记录在案

### 二、风险等级评定

#### 客户风险等级分类

| 风险等级 | 特征 | 管控措施 |
|----------|------|----------|
| 低风险 | 本地户籍、普通工薪族、交易简单 | 简化识别、常规监控 |
| 中风险 | 外地户籍、个体经营者、交易较频繁 | 标准识别、定期监控 |
| 高风险 | 跨境交易频繁、现金交易多、PEP | 增强识别、密切监控 |
| 禁止类 | 制裁名单、严重洗钱嫌疑 | 拒绝建立关系/终止关系 |

#### 风险评分维度

| 评分维度 | 权重 | 评估内容 |
|----------|------|----------|
| 客户特征 | 25% | 职业、国籍、年龄、地域 |
| 地域风险 | 20% | 户籍所在地、经营所在地 |
| 业务特征 | 25% | 产品类型、交易渠道、交易规模 |
| 行业风险 | 15% | 所属行业洗钱风险等级 |
| 行为特征 | 15% | 交易行为模式、配合度 |

### 三、持续识别

#### 持续识别触发条件

1. **定期更新**：
   - 低风险客户：每3年更新一次身份信息
   - 中风险客户：每2年更新一次
   - 高风险客户：每1年更新一次
   - 身份证件到期：证件到期前提醒更新

2. **交易触发**：
   - 单笔或日累计交易超过5万元（现金交易）
   - 单笔或日累计跨境交易超过1万元
   - 交易模式发生显著变化

3. **系统触发**：
   - 反洗钱系统发出预警
   - 监控发现可疑交易行为
   - 外部名单命中（制裁名单、关注名单）

#### 持续识别操作

1. 核对客户基本信息是否有变化（地址、职业、联系方式）
2. 更新身份证件有效期
3. 了解客户近期交易特征是否有异常
4. 重新评估客户风险等级
5. 记录识别结果

### 四、重新识别

#### 触发重新识别的情形

1. 客户身份证件过期未更新
2. 客户交易行为出现异常（大额频繁转账、突然改变交易模式）
3. 客户涉及负面新闻或司法案件
4. 反洗钱系统预警
5. 监管机构通报或协查
6. 客户被列入制裁名单或关注名单
7. 客户风险等级评估结果发生变化

#### 重新识别操作

1. 要求客户提供更新的身份证件
2. 补充收集辅助身份证明材料
3. 重新核实客户身份和实际控制人
4. 深入了解资金来源和交易目的
5. 重新评估风险等级
6. 必要时采取增强型尽职调查
7. 如发现可疑情况，提交可疑交易报告

### 五、增强型尽职调查（EDD）

#### 适用客户

- 政治公众人物（PEP）及其亲属
- 高风险国家/地区客户
- 现金密集型行业客户
- 交易规模与客户身份不匹配的客户
- 其他高风险客户

#### EDD内容

1. 深入了解客户财富和资金来源
2. 了解客户建立业务关系的目的和预期交易
3. 获取高级管理层审批方可建立业务关系
4. 加强交易监控频率和力度
5. 定期（至少每年）复核客户信息
6. 了解资金来源和资金去向

## 业务规则

1. **实名制**：必须使用有效身份证件实名开户，严禁匿名或假名开户
2. **拒绝情形**：客户拒绝提供身份信息或提供虚假信息的，银行有权拒绝建立业务关系
3. **保存期限**：客户身份资料保存至业务关系结束后至少5年
4. **信息更新**：客户身份信息变化时应及时到银行更新
5. **证件过期**：身份证件过期超过90天未更新的，银行可限制账户功能
6. **代理识别**：代理办理业务须同时识别代理人和被代理人身份
7. **一次性交易**：一次性现金交易5万元以上须进行客户身份识别
8. **不得泄露**：不得向客户泄露可疑交易报告等反洗钱工作信息

## 异常处理

### 场景一：联网核查不一致

- 确认身份证是否在有效期内、是否为原件
- 核查公安系统数据是否存在更新延迟
- 要求客户提供辅助身份证明（户口簿、驾驶证等）
- 如仍无法核实，联系公安机关确认
- 暂不办理业务，待身份确认后再办

### 场景二：客户拒绝提供信息

- 向客户解释KYC是法律法规要求
- 说明不提供信息的后果（无法办理业务）
- 如客户坚持不提供，拒绝建立业务关系
- 记录拒绝情况并上报

### 场景三：证件过期

- 提醒客户及时更新身份证件
- 设置90天过渡期，期间限制部分功能
- 过渡期结束仍未更新的，暂停非柜面交易
- 客户更新证件后恢复正常功能

### 场景四：发现客户在制裁名单上

- 立即暂停所有交易
- 冻结客户账户资金
- 上报总行反洗钱部门和合规部门
- 按规定向人民银行和公安机关报告
- 配合监管部门后续调查

## 风控要点

1. **合规底线**：KYC是反洗钱的基础要求，不得放松或变通
2. **实质重于形式**：不仅核实证件，还要了解客户的真实身份和交易目的
3. **风险为本**：根据客户风险等级差异化实施KYC
4. **信息安全**：客户身份信息严格保密，不得泄露
5. **持续有效**：KYC不是一次性工作，须持续更新和监控
6. **留痕管理**：所有识别过程和结果完整记录留痕
7. **培训要求**：一线员工须定期接受反洗钱和KYC培训

## 城商行/农商行特色

1. **熟人社会优势**：本地社区客户多为熟人，可通过社区关系辅助身份核实
2. **方言服务**：农村老年客户KYC提供方言服务，耐心解释身份识别的必要性
3. **上门核实**：对行动不便的客户或偏远农村客户，可上门进行身份核实
4. **村委协作**：借助村委会、社区力量辅助核实客户身份和背景
5. **系统局限**：部分中小银行KYC系统较为基础，需加强系统建设
6. **培训不足**：基层员工反洗钱意识参差不齐，需加强培训
7. **本地风险特征**：结合本地经济特征（如边贸、渔业等）制定差异化的KYC策略
