# 商业银行互联网贷款管理暂行办法

## 法规概述

| 项目 | 内容 |
|------|------|
| 法规全称 | 《商业银行互联网贷款管理暂行办法》 |
| 发布机构 | 原中国银行保险监督管理委员会 |
| 发布时间 | 2020年7月12日（银保监会令2020年第9号） |
| 补充文件 | 2021年2月《关于进一步规范商业银行互联网贷款业务的通知》、2022年7月《关于加强商业银行互联网贷款业务管理提升金融服务质效的通知》 |
| 文号 | 银保监会令2020年第9号 |
| 效力层级 | 部门规章 |

《商业银行互联网贷款管理暂行办法》是互联网贷款业务的首部专项监管规章，明确了互联网贷款的定义、经营模式、风险管理、消费者保护等方面的基本要求。后续发布的两个补充通知进一步细化了出资比例、集中度管理、跨区域经营等方面的规定。该办法对城商行、农商行的线上贷款业务发展具有重大指导意义。

## 核心内容摘要

### 一、互联网贷款的定义与分类

- **第2条**：互联网贷款是指商业银行运用互联网和移动通信等信息通信技术，基于风险数据和风险模型进行交叉验证和风险管理，线上自动完成贷款申请、风险评估、授信审批、合同签订、贷款发放等核心业务环节的贷款。
- **第8条**：互联网贷款按照经营模式分为自主经营贷款和联合贷款。

### 二、核心风控不得外包

- **第5条**：商业银行应当独立开展互联网贷款风险管理，并自主完成对贷款风险评估和风险控制具有重要影响的风控环节，严禁将贷前、贷中、贷后的关键风控环节外包。

### 三、联合贷款管理

- **第15条（2021年补充通知）**：商业银行与合作机构共同出资发放贷款时，单笔贷款中合作方出资比例不得低于**30%**。
- **第16条（2021年补充通知）**：商业银行与合作机构共同出资发放贷款的余额不得超过本行全部贷款余额的**50%**。
- **第17条（2022年补充通知）**：地方法人银行（城商行、农商行）开展互联网联合贷款业务的，不得跨区域经营。

### 四、集中度管理

- **第18条（2021年补充通知）**：商业银行与单一合作机构（含其关联方）共同出资发放的贷款余额不得超过本行全部贷款余额的**25%**。
- **第19条（2021年补充通知）**：商业银行与单一合作机构共同出资发放互联网贷款中，该合作机构提供的担保余额不得超过该机构净资产的**4倍**。

### 五、贷款限额

- **第10条**：单户用于消费的个人信用贷款授信额度不超过**20万元**，到期一次性还本的，授信期限不超过**1年**。
- 对于超过20万元或期限超过1年的互联网贷款，应当通过线下方式补充进行面签或面谈。

### 六、消费者保护

- **第23条**：商业银行应当在贷款合同中明确贷款利率、费用、还款方式等信息，不得强制搭售其他产品或服务。
- **第24条**：商业银行应当保护借款人信息安全，不得将借款人信息用于与贷款业务无关的目的。

## 零售业务相关条款解读

### （一）互联网贷款的业务模式

互联网贷款在零售业务中的应用场景主要包括：

**1. 自主经营贷款**

银行独立完成的线上贷款业务：
- 银行自主获客、自主审批、自主放款、自主催收
- 适用于标准化程度高的小额消费贷款
- 城商行、农商行可基于本地客户资源开发线上贷款产品
- 需要银行具备自主的风控模型和系统能力

**2. 联合贷款**

银行与合作机构共同出资发放贷款：
- 合作机构可以是其他商业银行、消费金融公司、互联网银行等
- 各方按照约定比例出资，共担风险
- 商业银行须对合作机构进行准入审查和持续管理
- 须严格执行出资比例和集中度要求

**3. 助贷模式**

合作机构仅提供获客和风控辅助，不出资：
- 助贷机构负责客户推荐和初步筛选
- 银行独立完成风险评估和授信审批
- 银行独立承担信用风险
- 助贷费用计入贷款成本

### （二）核心风控不得外包的具体要求

"核心风控不得外包"是互联网贷款监管的红线，具体包括：

**1. 贷前环节**

- 授信审批必须由银行独立完成
- 风险评估模型必须由银行自主开发或深度参与
- 客户准入标准必须由银行自主制定
- 不得将授信审批权让渡给合作机构

**2. 贷中环节**

- 贷款发放必须由银行独立决策
- 贷款合同必须由银行与借款人直接签订
- 贷款利率和费用必须由银行自主确定
- 资金划转必须由银行直接操作

**3. 贷后环节**

- 贷后管理必须由银行主导实施
- 风险预警必须由银行独立判断
- 贷款分类和拨备计提必须由银行自主完成
- 催收可以由第三方执行，但银行须承担管理责任

### （三）跨区域经营限制

2022年补充通知对地方法人银行（城商行、农商行）的互联网贷款跨区域经营做出了明确限制：

**1. 禁止跨区域经营**

- 地方法人银行开展互联网贷款业务的，应服务于当地客户
- 不得跨注册地辖区开展互联网贷款业务
- 无实体经营网点的互联网银行（如微众银行、网商银行）除外

**2. "当地客户"的认定标准**

- 户籍所在地在银行注册地辖区内
- 工作所在地在银行注册地辖区内
- 居住所在地在银行注册地辖区内
- 经常活动区域在银行注册地辖区内

**3. 存量业务的整改**

- 对于已存在的跨区域互联网贷款，应在过渡期内逐步清理
- 不得新增跨区域互联网贷款
- 过渡期结束后仍存续的，应纳入整改计划

### （四）个人消费贷款的限额管理

| 贷款类型 | 额度上限 | 期限限制 | 特殊要求 |
|----------|----------|----------|----------|
| 个人信用贷款（线上） | 20万元 | 到期一次还本不超过1年 | 超过需线下面签 |
| 个人信用贷款（线下补充面签） | 不限 | 按个人贷款管理办法 | 面谈面签 |
| 个人抵押贷款（线上） | 根据抵押物价值 | 按个人贷款管理办法 | 抵押登记手续 |
| 个人经营贷款（线上） | 根据经营情况 | 按个人贷款管理办法 | 经营真实性核实 |

### （五）合作机构管理

**1. 合作机构准入**

- 建立合作机构准入标准和审批流程
- 对合作机构的资质、经营状况、风控能力进行全面评估
- 签订合作协议，明确各方权利义务
- 建立合作机构名单管理制度

**2. 持续监控**

- 定期对合作机构进行评估和检查
- 监控合作机构的出资比例、集中度等指标
- 建立合作机构退出机制
- 对合作机构的违规行为及时采取管控措施

**3. 信息披露**

- 在贷款合同中明确披露合作机构信息
- 向借款人说明各方在贷款中的角色和责任
- 公示合作机构名单

## 对城商行/农商行的影响

### 1. 线上贷款业务发展的重大约束

跨区域经营限制对城商行、农商行的影响最为直接：
- 无法通过互联网贷款实现全国范围的规模扩张
- 线上贷款业务只能服务于当地客户
- 需要重新审视和调整线上贷款业务战略
- 部分以互联网贷款为主要增长动力的银行面临重大转型压力

### 2. 联合贷款模式的调整

- 出资比例要求（合作方不低于30%）影响了联合贷款的经济性
- 集中度管理（单一合作方不超过25%）限制了与单一平台的深度合作
- 需要拓展多元化的合作机构，分散合作集中度风险
- 逐步提升自主风控和自主获客能力

### 3. 自主风控能力建设

核心风控不得外包的要求促使城商行、农商行加快自主风控能力建设：
- 组建专业的数据分析和风控建模团队
- 建设自主的大数据风控平台
- 积累和分析本地客户的行为数据
- 开发适合本地市场的线上贷款产品

### 4. 数字化转型的机遇

互联网贷款办法虽然带来了约束，但也推动了城商行、农商行的数字化转型：
- 倒逼银行加快IT系统建设
- 促进银行引入数字化人才
- 推动银行探索线上线下融合的服务模式
- 激发银行利用本地数据优势创新产品

## 合规实施要点

### 1. 制度建设

| 制度名称 | 关键内容 |
|----------|----------|
| 互联网贷款管理办法实施细则 | 业务流程、风控标准、操作规范 |
| 互联网贷款合作机构管理办法 | 准入标准、持续管理、退出机制 |
| 互联网贷款风险管理制度 | 风控模型管理、数据安全、贷后管理 |
| 互联网贷款集中度管理办法 | 比例监控、预警机制、压降计划 |
| 互联网贷款跨区域经营管理办法 | 客户认定标准、地理围栏技术、合规监控 |
| 互联网贷款消费者权益保护办法 | 息费披露、信息安全、投诉处理 |

### 2. 系统建设

- **互联网贷款平台**：贷款申请、在线审批、电子签约、放款管理
- **风控决策引擎**：反欺诈、信用评分、规则引擎、模型管理
- **合作机构管理系统**：准入管理、出资比例监控、集中度监控
- **地理围栏系统**：客户属地判断、跨区域预警
- **数据中台**：内外部数据整合、特征工程、模型训练

### 3. 人员配置

- 配备专业的互联网贷款产品经理
- 建设数据分析和风控建模团队
- 配置互联网贷款运营管理团队
- 设立专职的互联网贷款合规管理岗位

## 监管处罚案例

### 案例一：某城商行互联网贷款核心风控外包案（2023年）

- **违规事实**：该城商行在与某互联网平台合作发放联合贷款时，将授信审批环节实质上让渡给了合作平台，银行仅进行形式审查即放款。风控模型完全由合作平台提供，银行未参与模型开发和验证。
- **处罚结果**：被罚款400万元，暂停联合贷款新业务6个月。
- **涉及条款**：第5条（核心风控不得外包）。

### 案例二：某农商行互联网贷款跨区域经营案（2022年）

- **违规事实**：该农商行通过互联网贷款平台向全国多个省份的客户发放个人消费贷款，跨区域贷款余额达到12亿元，严重违反了地方法人银行不得跨区域经营的规定。
- **处罚结果**：被罚款300万元，要求限期清退跨区域贷款。
- **涉及条款**：2022年补充通知（跨区域经营限制）。

### 案例三：某城商行互联网贷款集中度超标案（2023年）

- **违规事实**：该城商行与单一互联网平台合作发放的联合贷款余额占全部贷款余额的42%，远超25%的集中度上限。
- **处罚结果**：被罚款200万元，要求限期压降至合规水平。
- **涉及条款**：第18条（集中度管理）。

### 案例四：某银行互联网贷款信息安全案（2022年）

- **违规事实**：某银行在与合作机构共享客户数据时，未对借款人信息进行脱敏处理，导致借款人个人信息泄露。
- **处罚结果**：被罚款150万元。
- **涉及条款**：第24条（借款人信息安全）。
