# 05 - 渠道服务功能

## 分类概述

渠道服务功能是零售银行业务知识库中规模最大的分类，涵盖银行通过各类触达渠道向零售客户提供服务的全部功能体系。本分类按照渠道类型和业务板块进行系统化组织，旨在为产品经理、业务人员和开发人员提供完整的渠道服务参考。

## 渠道体系架构

城商行/农商行的零售渠道体系呈现"线上+线下"双轨并行的格局：

### 线上渠道

| 渠道类型 | 核心定位 | 覆盖场景 | 文档数量 |
|---------|---------|---------|---------|
| 手机银行 | 零售主阵地，承载80%以上线上交易 | 全业务覆盖 | 约45篇 |
| 网上银行 | 大额交易与复杂操作的补充渠道 | 转账、理财、查询 | 3篇 |
| 微信银行 | 轻量触达，高频互动 | 动账通知、快捷查询 | 3篇 |
| 远程银行 | 非面对面服务，覆盖长尾客户 | 咨询、面签、远程办理 | 4篇 |

### 线下渠道

| 渠道类型 | 核心定位 | 覆盖场景 | 文档数量 |
|---------|---------|---------|---------|
| 智能柜台（STM） | 柜面业务迁移，降低人力成本 | 开卡、签约、挂失等 | 3篇 |
| ATM自助设备 | 现金存取与基础交易 | 存取款、转账、查询 | 含在自助设备中 |
| 柜面服务 | 复杂业务与适老化兜底 | 全业务，尤其大额现金 | 3篇 |

### 生态渠道

| 渠道类型 | 核心定位 | 覆盖场景 | 文档数量 |
|---------|---------|---------|---------|
| 开放银行 | API输出，嵌入第三方场景 | 账户、支付、数据服务 | 3篇 |

## 文档总计

## 编写规范

每篇文档统一采用以下结构：

- **功能清单类**：功能列表（表格）+ 功能说明 + 办理渠道 + 业务限制
- **业务规则类**：规则清单 + 限额/费用表 + 合规要求 + 异常处理
- **操作流程类**：前置条件 + 分步操作（编号步骤）+ 注意事项 + 异常处理
- **常见问题类**：FAQ列表 + 风控要点 + 典型案例
