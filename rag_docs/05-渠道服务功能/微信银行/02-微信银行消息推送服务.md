# 微信银行消息推送服务

> 本文档详细介绍基于微信模板消息和订阅消息的银行通知服务体系，涵盖消息类型、推送规则、模板设计以及与短信通知的对比分析。

---

## 一、概述

微信银行消息推送服务是银行通过微信公众号的模板消息/订阅消息接口，向已关注并绑定账户的客户主动推送金融通知的服务。该服务依托微信的高打开率和即时性，已成为替代或补充传统短信通知的重要客户触达渠道。

根据微信平台的规则演进，消息推送经历了三个阶段：
- **模板消息**（2014-2021年）：服务号可向用户推送预定义模板格式的消息。
- **订阅通知**（2021年至今）：需用户主动订阅授权后方可推送，尊重用户选择权。
- **一次性订阅消息**：用户每次点击"允许"后可推送一条消息。

---

## 二、消息推送类型

### （一）动账提醒

最核心、最高频的推送类型，覆盖所有资金变动场景：

| 消息场景 | 模板内容 | 触发条件 |
|---------|---------|---------|
| 账户入账 | 交易类型、入账金额、账户余额、交易时间、交易对手 | 账户收到转账、工资入账等 |
| 账户出账 | 交易类型、出账金额、账户余额、交易时间、收款人 | 转账、消费、缴费等支出 |
| 理财购买成功 | 产品名称、购买金额、预期收益率、起息日 | 理财产品购买确认 |
| 理财赎回成功 | 产品名称、赎回金额、到账日期 | 理财产品赎回到账 |

### （二）信用卡还款提醒

| 消息场景 | 模板内容 | 推送时间 |
|---------|---------|---------|
| 账单生成通知 | 本期账单金额、最低还款额、到期还款日 | 账单日后1天内 |
| 还款日提醒 | 应还金额、还款日、快捷还款入口 | 还款日前3天、前1天各推送一次 |
| 还款成功确认 | 还款金额、还款时间、剩余额度 | 还款入账后即时推送 |
| 逾期提醒 | 逾期金额、违约金、最低还款额、催收提示 | 逾期后即时推送 |

### （三）理财到期提醒

| 消息场景 | 模板内容 | 推送时间 |
|---------|---------|---------|
| 理财即将到期 | 产品名称、到期日期、本金金额、预期收益 | 到期前3天 |
| 理财到期兑付 | 产品名称、兑付金额、实际收益、到账状态 | 到期当日或资金到账时 |
| 定存到期提醒 | 存单金额、到期日期、续存建议 | 到期前7天、前1天 |

### （四）贷款还款提醒

| 消息场景 | 模板内容 | 推送时间 |
|---------|---------|---------|
| 还款日提醒 | 贷款类型、应还金额、还款日、扣款账户 | 还款日前5天、前1天 |
| 扣款成功通知 | 扣款金额、剩余本金、下期还款日 | 扣款成功后即时推送 |
| 逾期预警 | 逾期天数、逾期金额、罚息、征信影响提示 | 逾期后第1天、第3天、第7天 |
| 结清通知 | 贷款类型、结清日期、总还款金额 | 贷款结清后即时推送 |

### （五）营销活动推送

| 消息场景 | 模板内容 | 推送频率 |
|---------|---------|---------|
| 新活动上线 | 活动名称、活动简介、参与方式 | 每周不超过2次 |
| 信用卡优惠 | 优惠商户、折扣力度、有效期 | 每周不超过1次 |
| 专属产品推荐 | 产品名、收益率/利率、限时优惠 | 根据客户画像精准推送 |
| 活动即将结束 | 活动名称、剩余时间、参与入口 | 活动结束前1天 |

## 三、消息模板设计

### （一）模板结构

微信模板消息采用预定义的JSON格式，包含以下要素：

```
模板名称：账户资金变动通知
模板编号：OPENTM123456789

{{first.DATA}}            ← 消息标题
交易类型：{{keyword1.DATA}}
交易金额：{{keyword2.DATA}}
账户余额：{{keyword3.DATA}}
交易时间：{{keyword4.DATA}}
{{remark.DATA}}           ← 备注/操作指引
```

### （二）设计原则

1. **信息完整**：包含交易核心要素（金额、时间、余额），减少客户另行查询。
2. **简洁清晰**：避免冗长文字，关键数据用数字展示。
3. **安全提示**：末尾添加"如非本人操作，请立即拨打客服热线XXX"的安全提醒。
4. **操作引导**：提供"查看详情"链接，引导客户进入H5页面或小程序查看完整信息。
5. **品牌一致**：标题和备注中体现银行品牌名称。

## 四、推送规则与限制

### （一）微信平台规则

| 规则项 | 具体要求 |
|-------|---------|
| 用户授权 | 客户须关注公众号并完成账户绑定；订阅消息需客户主动点击授权 |
| 推送频率 | 模板消息无明确频率限制，但应避免过度推送引起用户投诉/取关 |
| 推送时间 | 建议工作时段推送（8:00-21:00），避免深夜打扰 |
| 消息类型 | 仅限服务通知类，不得推送纯广告内容 |
| 链接跳转 | 仅支持跳转本行域名下的H5页面或小程序 |

### （二）银行内部管控

1. **免打扰设置**：客户可在公众号内设置消息免打扰时段或关闭特定类型通知。
2. **分级推送**：根据客户等级和偏好差异化推送内容和频率。
3. **去重机制**：同一客户短时间内多笔交易时，合并推送或限制推送频率。
4. **退订机制**：客户可通过回复"TD"或在公众号设置中取消订阅。

## 五、与短信通知的对比

| 对比维度 | 微信消息推送 | 短信通知 |
|---------|------------|---------|
| **触达率** | 高（微信高频使用，消息易被看到） | 中（短信打开率逐年下降） |
| **推送成本** | 极低（微信平台免费接口） | 较高（0.03-0.05元/条） |
| **信息容量** | 丰富（结构化模板，支持链接跳转） | 有限（70个汉字/条） |
| **交互性** | 强（可跳转H5/小程序直接操作） | 弱（单向通知） |
| **覆盖范围** | 仅覆盖微信用户且关注公众号的客户 | 覆盖所有手机号用户 |
| **安全性** | 中高（依赖微信安全体系） | 较高（独立通道，不依赖第三方平台） |
| **适用客群** | 年轻客群、高频微信用户 | 全客群（尤其老年客户、非智能手机用户） |
| **监管要求** | 需符合微信平台规则 | 需符合工信部短信管理规定 |

### 最佳实践建议

- **双通道并行**：对重要动账通知（大额交易、逾期提醒等）同时通过微信和短信推送，确保触达。
- **客户自选**：在账户签约时让客户选择通知渠道偏好（微信/短信/两者兼收）。
- **成本优化**：对小额动账通知优先使用微信推送，降低短信成本。
- **合规兜底**：涉及征信、逾期等法律告知义务的通知，必须使用短信渠道。

---

## 六、附注

- 微信消息推送功能受微信平台政策影响较大，需持续关注接口规则变化。
- 推送内容不得包含客户完整卡号、完整身份证号等敏感信息，应做脱敏处理。
- 建议定期分析消息打开率和客户投诉数据，优化推送策略和内容设计。
