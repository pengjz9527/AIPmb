# 手机银行安全认证体系

## 一、文档说明

本文档详细说明手机银行的安全认证体系，包括登录认证、交易认证、设备绑定三大核心安全机制，为产品设计和技术实现提供安全规范参考。

## 二、认证体系总体架构

手机银行安全认证体系遵循"分级认证、风险适配"原则，根据操作风险等级匹配不同强度的认证方式。

### 2.1 认证方式分级

| 认证级别 | 认证方式 | 安全强度 | 适用场景 |
|---------|---------|---------|---------|
| L1-基础认证 | 登录密码 | 低 | 登录、查询类操作 |
| L2-增强认证 | 短信验证码 | 中 | 小额转账、一般业务办理 |
| L3-强认证 | 动态令牌/云证通 | 高 | 大额转账、敏感操作 |
| L4-最强认证 | 人脸识别 | 最高 | 密码重置、设备解绑、开户 |

### 2.2 认证触发规则

| 操作类型 | 所需认证 | 说明 |
|---------|---------|------|
| 登录 | L1（密码）或生物识别 | 生物识别为密码登录的便捷替代 |
| 账户查询 | L1 | 仅需登录态 |
| 行内转账（≤5万） | L1 + L2 | 登录态+短信验证 |
| 行内转账（>5万） | L1 + L3 | 登录态+动态令牌 |
| 跨行转账（≤5万） | L1 + L2 | 登录态+短信验证 |
| 跨行转账（>5万） | L1 + L3 | 登录态+动态令牌 |
| 修改登录密码 | L1 + L4 | 旧密码+人脸识别 |
| 重置登录密码 | L4 | 人脸识别+身份信息核验 |
| 修改交易密码 | L1 + L3 + L4 | 登录态+令牌+人脸 |
| 设备解绑 | L1 + L4 | 密码+人脸识别 |

## 三、登录认证详解

### 3.1 密码登录

**功能说明**：客户通过输入登录密码完成身份验证，是最基础的认证方式。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 密码长度 | 8-20位字符 |
| 密码复杂度 | 必须包含字母+数字，不允许纯数字或纯字母 |
| 密码有效期 | 90天强制修改（可配置） |
| 连续错误锁定 | 连续输错5次，锁定30分钟 |
| 累计错误锁定 | 当日累计输错10次，锁定至次日0点 |
| 密码锁定解锁 | 柜面解锁或通过人脸识别自助解锁 |
| 密码修改历史 | 不得与前5次使用的密码相同 |

**安全增强措施**：

1. **安全键盘**：自定义随机键盘，按键位置每次随机排列，防止键盘钩子攻击
2. **密码加密传输**：密码在客户端RSA加密后传输，服务端不解密直接比对
3. **输入保护**：密码输入框禁止截屏、禁止第三方输入法（可配置）
4. **环境检测**：检测是否在Root/越狱设备上运行，若是则提示风险

### 3.2 指纹登录

**功能说明**：利用手机指纹传感器进行身份验证，作为密码登录的便捷替代。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 开启条件 | 已设置密码登录，且设备支持指纹识别 |
| 首次开启 | 需输入登录密码验证后绑定 |
| 回退机制 | 指纹识别失败3次后回退至密码登录 |
| 安全限制 | 设备重启后首次登录必须使用密码 |
| 有效期 | 每72小时需使用一次密码登录（安全兜底） |
| 指纹变更 | 检测到设备新增指纹时，需重新验证密码 |

### 3.3 面容登录（Face ID/面部识别）

**功能说明**：利用手机3D结构光或摄像头进行面部识别，实现便捷登录。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 设备要求 | iOS：Face ID设备；Android：支持3D结构光或红外活体检测的设备 |
| 活体检测 | 必须通过活体检测，防止照片/视频欺骗 |
| 环境要求 | 光线充足，面部无遮挡 |
| 安全等级 | 与指纹登录等同 |
| 失败回退 | 连续3次识别失败，回退至密码登录 |

### 3.4 手势密码登录

**功能说明**：通过绘制预设手势图案完成登录，适用于快速查看场景。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 手势复杂度 | 至少连接4个点，不可重复经过同一点 |
| 功能限制 | 手势登录仅支持查询类操作，交易操作仍需密码 |
| 错误锁定 | 连续5次错误，手势功能锁定，需密码登录后重新设置 |
| 有效期 | 每7天需使用一次完整密码登录 |

## 四、交易认证详解

### 4.1 短信验证码

**功能说明**：向客户预留手机号发送一次性验证码，用于交易确认。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 验证码长度 | 6位数字 |
| 有效期 | 5分钟（可配置3-10分钟） |
| 发送频率 | 同一手机号60秒内仅发送1次 |
| 日发送上限 | 单手机号每日最多20条 |
| 错误次数限制 | 连续输错5次，当日短信验证功能锁定 |
| 验证码防泄露 | 短信内容不显示具体交易金额，仅显示验证码和银行名称 |

**安全风险防范**：

1. **SIM卡劫持防护**：检测到手机号近期有SIM卡更换记录时，提升认证等级
2. **GSM劫持防护**：监测短信接收异常（突然无信号），提示用户确认
3. **社会工程防护**：短信中明确提示"银行工作人员不会索要验证码"

### 4.2 动态令牌（OTP Token）

**功能说明**：基于时间同步的一次性密码生成器，分为硬件令牌和软件令牌（云证通）。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 令牌类型 | 硬件令牌（实体设备）/ 云证通（手机软件令牌） |
| 密码长度 | 6位数字 |
| 刷新周期 | 每60秒自动刷新 |
| 绑定规则 | 一个令牌绑定一个客户号，不可共享 |
| 申请方式 | 硬件令牌需柜面领取；云证通可在线开通 |
| 失效处理 | 令牌丢失需柜面挂失并重新绑定 |
| 时间校准 | 令牌与服务器时间偏差超过阈值时自动校准 |

### 4.3 人脸识别认证

**功能说明**：通过手机摄像头采集人脸图像，与公安部人口库照片比对，完成身份验证。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 比对来源 | 公安部人口信息库（身份证照片） |
| 活体检测 | 必须通过动作活体+光线活体双重检测 |
| 识别阈值 | 相似度≥80%通过（可配置） |
| 尝试次数 | 单次操作最多3次尝试 |
| 失败处理 | 3次失败后降级为柜面办理 |
| 环境要求 | 光线充足、面部无遮挡、正对摄像头 |
| 数据保护 | 人脸图像不在客户端存储，传输后立即销毁 |

## 五、设备绑定管理

### 5.1 设备绑定机制

**功能说明**：将手机银行账号与特定移动设备绑定，限制非绑定设备的使用权限。

**业务规则**：

| 规则项 | 规则内容 |
|-------|---------|
| 最大绑定数 | 每个客户最多绑定3台设备 |
| 首次绑定 | 注册成功后自动绑定当前设备 |
| 新设备登录 | 新设备首次登录需短信验证+人脸识别 |
| 设备解绑 | 可在已绑定设备上自助解绑，或通过柜面解绑 |
| 全部解绑 | 所有设备解绑后，需重新通过人脸识别绑定新设备 |
| 设备信息展示 | 显示设备型号、绑定时间、最近登录时间和地点 |

### 5.2 设备指纹技术

**采集维度**：

| 维度类别 | 采集信息 |
|---------|---------|
| 硬件特征 | 设备型号、系统版本、屏幕分辨率、IMEI/IDFA（脱敏） |
| 网络环境 | IP地址、Wi-Fi SSID、基站信息、GPS定位（授权后） |
| 行为特征 | 打字速度、触摸力度、操作习惯、常用功能路径 |
| 环境特征 | 是否Root/越狱、是否安装Hook框架、是否使用模拟器 |

### 5.3 异常设备处理

| 异常场景 | 处理方式 |
|---------|---------|
| 新设备登录 | 触发增强认证（短信+人脸），验证通过后自动绑定 |
| 异地登录 | 弹出二次验证，同时发送短信通知已绑定设备 |
| 模拟器检测 | 拒绝登录，提示"请使用真实设备" |
| Root/越狱检测 | 允许登录但弹出安全警告，限制大额交易 |
| 多设备同时在线 | 踢出旧设备会话，保留最新登录设备 |

## 六、合规与监管要求

### 6.1 监管依据

1. **《商业银行信息科技风险管理指引》**：要求建立完善的身份认证体系
2. **《银行业金融机构信息系统安全管理规范》**：明确多因素认证要求
3. **《移动金融客户端应用软件安全管理规范》**（JR/T 0092）：明确客户端安全要求
4. **《个人信息保护法》**：生物识别信息属于敏感个人信息，需单独授权

### 6.2 安全审计要求

| 审计项 | 要求 |
|-------|------|
| 登录日志 | 记录每次登录的时间、设备、IP、认证方式、结果 |
| 交易日志 | 记录每笔交易的完整认证链路 |
| 异常事件 | 记录所有认证失败、风控触发、设备异常事件 |
| 日志保存 | 至少保存5年，支持审计查询 |
| 定期评估 | 每年至少进行一次安全评估和渗透测试 |

## 七、风控联动

### 7.1 实时风控规则

| 风控规则 | 触发条件 | 处置动作 |
|---------|---------|---------|
| 异地登录 | 登录IP与常用地址偏差>200km | 增强认证+短信通知 |
| 频繁登录 | 10分钟内登录失败≥3次 | 临时锁定+短信通知 |
| 大额交易 | 单笔交易>5万元 | 要求动态令牌认证 |
| 夜间交易 | 23:00-6:00发起交易 | 增强认证+风险提示 |
| 新设备交易 | 绑定新设备24小时内交易 | 限额降至5000元/日 |
| 高频交易 | 1小时内转账≥10笔 | 触发人工审核 |

### 7.2 风控处置流程

1. **风险识别**：风控引擎实时分析交易特征，识别风险信号
2. **风险评级**：根据风险信号组合评定风险等级（低/中/高/极高）
3. **风险处置**：低风险放行，中风险增强认证，高风险拦截，极高风险冻结账户
4. **事后分析**：定期回顾风控事件，优化规则参数，降低误报率
