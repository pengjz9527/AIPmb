# 手机银行 — 生活缴费与支付业务规则

## 一、缴费业务通用规则

### 1.1 缴费渠道与账户限制

| 规则项 | 规则说明 |
|--------|----------|
| 缴费账户类型 | 支持本行借记卡（I类户/II类户）、信用卡（部分缴费项目不支持信用卡） |
| 账户状态要求 | 缴费账户须为正常状态，冻结、挂失、过期账户不可用于缴费 |
| II类户限制 | II类户缴费受日累计1万元、年累计20万元的出金限额约束 |
| 缴费币种 | 仅支持人民币（CNY）缴费 |
| 跨行缴费 | 部分缴费项目支持跨行卡缴费（需接入银联/网联通道），具体以页面展示为准 |
| 第三方代扣 | 客户可授权我行从指定账户自动扣缴，需签署《委托代扣协议》 |

### 1.2 缴费时间与到账规则

| 规则项 | 规则说明 |
|--------|----------|
| 缴费服务时间 | 一般缴费项目7×24小时可操作；部分项目受第三方机构系统窗口限制 |
| 到账时效 | 实时缴费：提交后1-5分钟内到账（水电气等主流项目） |
| 非实时缴费 | 部分项目T+1日到账（如学费、物业费等对账类项目） |
| 缴费确认 | 缴费结果以第三方机构系统确认结果为准，非以我行扣款时间为准 |
| 重复缴费 | 系统对同一户号、同一账期的缴费请求进行幂等校验，防止重复扣款 |
| 欠费查询 | 部分缴费项目支持欠费金额实时查询，客户可确认缴费金额后再提交 |

## 二、对账规则

### 2.1 日终对账机制

| 对账类型 | 对账频率 | 对账方式 | 差异处理 |
|----------|----------|----------|----------|
| 行内对账 | 每日日终 | 核心系统交易流水与手机银行交易记录核对 | 差异笔数自动标记，T+1日人工核实 |
| 渠道对账 | 每日日终 | 我行系统与第三方机构（水务/电力/燃气等）系统对账 | 以第三方机构确认为准，差异3个工作日内处理 |
| 资金对账 | 每日日终 | 我行清算账户与第三方清算资金核对 | 资金差异由运营管理部牵头处理 |
| 银联/网联对账 | 每日日终 | 通过银联/网联平台完成跨行交易对账 | 差错交易按银联差错处理流程处理 |

### 2.2 对账异常处理

1. **长款处理**（我行有记录、第三方无记录）：T+1日核实后，3个工作日内发起退款或补交易。
2. **短款处理**（第三方有记录、我行无记录）：T+1日核实后，补录交易或联系客户补缴。
3. **金额不一致**：以第三方机构账单金额为准，差额部分由运营管理岗协调处理。
4. **对账文件缺失**：若第三方未在约定时间提供对账文件，触发告警并人工跟进。

## 三、退款规则

### 3.1 退款场景与规则

| 退款场景 | 退款规则 | 退款时效 | 退款方式 |
|----------|----------|----------|----------|
| 缴费失败（扣款成功但第三方未确认） | 系统自动发起退款 | 1-5个工作日原路退回 | 自动退款 |
| 重复缴费 | 客户申请退款，核实后退回多缴金额 | 5-10个工作日 | 人工审核后退款 |
| 多缴金额 | 超出实际欠费的金额自动退回 | 以第三方机构退费周期为准（通常10-15个工作日） | 自动退款 |
| 话费充值失败 | 充值未到账，系统核实后自动退款 | 1-3个工作日 | 自动退款 |
| 交通罚款缴纳失败 | 罚款未成功入库，自动退款 | 3-5个工作日 | 自动退款 |
| 客户主动申请退款 | 缴费成功后原则上不支持退款；特殊情况需柜面处理 | 需第三方机构确认后处理 | 柜面办理 |

### 3.2 退款特殊说明

- **不可退款场景**：话费已充值到账、罚款已入库、学费已入账等情况，我行不直接办理退款，客户需联系对应机构处理。
- **退款资金来源**：退款原路返回至原缴费账户，若原账户已注销，需客户至柜面提供新账户办理。
- **退款通知**：退款完成后通过APP消息推送及短信通知客户。

## 四、支付限额规则

### 4.1 扫码支付限额

| 支付场景 | 单笔限额 | 日累计限额 | 月累计限额 | 认证方式 |
|----------|----------|------------|------------|----------|
| 主扫支付（C扫B） | 5万元 | 20万元 | 100万元 | 交易密码/生物识别 |
| 被扫支付（B扫C）免密 | 1000元 | 3000元 | — | 无需验证 |
| 被扫支付（B扫C）验密 | 5万元 | 20万元 | 100万元 | 交易密码 |
| 收付款码 | 5万元 | 20万元 | 100万元 | 交易密码/生物识别 |
| 二维码转账（个人间） | 5万元 | 20万元 | 100万元 | 交易密码+短信 |
| NFC手机闪付（免密） | 1000元 | 3000元 | — | 无需验证 |
| NFC手机闪付（验密） | 5万元 | 20万元 | 100万元 | 交易密码/生物识别 |

### 4.2 限额调整规则

1. **客户自主调低**：客户可在手机银行"支付限额管理"中自主下调单笔/日累计限额，即时生效。
2. **限额上调**：需通过柜面或远程银行（视频面签）进行身份核验后方可上调，上调后T+1日生效。
3. **免密支付限额**：免密支付限额不可上调，客户可在设置中关闭免密支付功能。
4. **风险控制**：当系统监测到异常交易时，可临时降低客户限额或暂停扫码支付功能，并通过短信/APP通知客户。

## 五、缴费业务风控规则

### 5.1 交易监控规则

| 风控场景 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 高频缴费 | 同一账户1小时内缴费超过10笔 | 触发二次身份验证（短信+交易密码） |
| 大额缴费 | 单笔缴费超过3万元 | 强制短信验证码+交易密码双重认证 |
| 异地缴费 | 缴费IP/GPS定位与常住城市不符 | 提示风险，需客户确认后方可继续 |
| 异常户号 | 缴费户号不在客户常用列表中且金额超1万元 | 增加交易密码验证 |
| 夜间交易 | 23:00-06:00期间缴费金额超5000元 | 增加短信验证码验证 |
| 设备异常 | 非常用设备发起缴费 | 增加人脸识别验证 |

### 5.2 反洗钱与合规要求

1. **客户身份识别**：首次使用缴费功能时进行身份信息确认，大额缴费需强化身份验证。
2. **可疑交易报告**：系统自动监测可疑交易模式（如频繁小额测试、关联账户集中缴费等），符合可疑交易特征的自动生成报告。
3. **缴费用途合规**：缴费资金须为客户自有资金，不得利用缴费渠道进行套现、洗钱等违法活动。

## 六、自动代扣业务规则

| 规则项 | 规则说明 |
|--------|----------|
| 签约方式 | 手机银行线上签约或柜面签约，需签署《自动代扣授权协议》 |
| 代扣触发 | 每月账单生成后自动扣款，扣款日一般为账单日后3个工作日内 |
| 代扣失败 | 余额不足时，系统在3个工作日内重试最多3次，均失败后发送缴费提醒通知 |
| 代扣限额 | 单笔代扣金额不超过账户可用余额，且不适用免密限额 |
| 代扣撤销 | 客户可随时在手机银行撤销代扣授权，撤销后次月起不再自动扣款 |
| 代扣通知 | 每次代扣成功后通过APP推送和短信通知客户 |

## 七、扫码支付特殊规则

### 7.1 二维码安全规则

1. **动态码有效期**：付款码每60秒自动刷新，旧码立即失效。
2. **一次性使用**：每个付款码仅可使用一次，使用后立即失效。
3. **码牌验证**：扫描商户静态码时，APP展示商户名称供客户确认，防止码牌替换欺诈。
4. **截屏防护**：付款码页面禁止截屏（Android）/截屏时发出警告（iOS）。

### 7.2 NFC支付规则

1. **设备要求**：手机须支持NFC功能且已开通手机闪付服务。
2. **绑卡验证**：首次绑卡需通过短信验证+交易密码验证。
3. **脱机交易**：NFC支付在脱机模式下完成，交易数据后续同步至我行系统。
4. **终端兼容**：支持所有具备银联闪付标识的POS终端。

## 八、争议处理规则

| 争议类型 | 处理时效 | 处理流程 |
|----------|----------|----------|
| 缴费未到账 | 提交后3个工作日内初步回复 | 客户提交争议申请→客服登记→业务部门核实→反馈处理结果 |
| 重复扣款 | 确认后5个工作日内退款 | 系统核实→确认重复扣款→发起退款→通知客户 |
| 扫码支付争议 | 提交后5个工作日内处理 | 客户提交争议→调取交易明细→联系商户核实→协商处理 |
| 退款未到账 | 超时后自动跟进 | 系统监控退款状态→超时触发催办→通知客户进度 |
