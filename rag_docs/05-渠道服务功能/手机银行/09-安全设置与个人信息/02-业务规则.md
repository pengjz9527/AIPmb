# 手机银行 — 安全设置与个人信息业务规则

## 一、密码管理规则

### 1.1 登录密码规则

| 规则项 | 规则说明 |
|--------|----------|
| 密码长度 | 8-20位字符 |
| 复杂度要求 | 必须包含大小写字母、数字、特殊字符中至少3种组合 |
| 禁止规则 | 不得与最近5次使用过的密码相同；不得包含连续4位以上相同字符；不得包含手机号、身份证号等个人信息 |
| 有效期 | 建议每90天修改一次；系统将在密码使用满180天时提醒修改 |
| 错误次数限制 | 连续输错5次锁定账户30分钟；连续输错10次锁定至当日24时；连续输错15次需至柜面解锁 |
| 锁定解锁 | 30分钟临时锁定到期后自动解锁；当日锁定需次日自动解锁或至柜面办理解锁 |
| 修改频率 | 同一密码24小时内仅可修改1次 |

### 1.2 交易密码规则

| 规则项 | 规则说明 |
|--------|----------|
| 密码格式 | 6位纯数字 |
| 禁止规则 | 不得为6位相同数字（如111111）；不得为连续数字（如123456/654321）；不得为生日等易猜数字 |
| 错误次数限制 | 连续输错3次锁定交易功能；锁定后需通过人脸识别解锁或至柜面解锁 |
| 修改验证 | 修改交易密码需验证原交易密码+短信验证码 |
| 重置方式 | 交易密码重置必须通过柜面办理或远程银行视频面签办理 |
| 修改频率 | 同一交易密码24小时内仅可修改1次 |

### 1.3 密码安全策略

1. **首次登录强制修改**：客户首次登录手机银行或柜面初始密码登录后，系统强制要求修改登录密码。
2. **密码不可逆存储**：所有密码以加密形式存储，系统不保存明文密码，银行工作人员无法查看客户密码。
3. **密码传输加密**：密码在传输过程中采用国密SM2/SM4加密算法，确保传输安全。
4. **键盘安全**：密码输入使用安全键盘（随机键盘/乱序键盘），防止键盘记录和截屏窃取。
5. **密码提示**：系统不存储密码提示信息，忘记密码只能通过重置流程处理。

## 二、生物识别安全规则

### 2.1 指纹识别规则

| 规则项 | 规则说明 |
|--------|----------|
| 开通条件 | 手机支持指纹识别 + 已完成手机银行注册 + 已通过身份验证 |
| 开通方式 | 手机银行"安全设置"→"生物识别"→"指纹登录/支付"→验证登录密码+交易密码→采集当前指纹→开通成功 |
| 指纹数量 | 最多注册3个指纹（建议注册不同手指） |
| 支付限额 | 指纹支付单笔限额5万元，日累计限额20万元 |
| 降级策略 | 指纹识别失败3次后自动降级为密码验证 |
| 安全要求 | 手机重启后首次登录必须使用密码验证，之后方可使用指纹 |
| 关闭方式 | 手机银行"安全设置"中关闭或柜面关闭 |

### 2.2 面容识别规则

| 规则项 | 规则说明 |
|--------|----------|
| 开通条件 | 手机支持3D结构光/ToF面容识别 + 已完成手机银行注册 |
| 安全等级 | 面容识别安全等级高于指纹识别（误识率低于1/1,000,000） |
| 支付限额 | 面容支付单笔限额10万元，日累计限额50万元 |
| 活体检测 | 开启面容支付时默认启用活体检测，防止照片/视频攻击 |
| 环境要求 | 面容识别需在光线充足环境下使用，强光/暗光环境可能影响识别 |
| 降级策略 | 面容识别失败2次后自动降级为密码验证 |
| 口罩识别 | 部分设备支持口罩识别模式（iOS 15.4+），但安全等级略低于完整面容 |

### 2.3 生物识别通用规则

1. **生物特征不上传**：指纹和面容数据仅存储在设备安全芯片（TEE/SE）中，不上传至银行服务器。
2. **设备绑定**：生物识别与设备绑定，更换设备后需重新开通。
3. **多用户限制**：同一手机银行账号仅可在1台设备上启用生物识别。
4. **异常锁定**：系统检测到生物识别异常使用（如频繁失败、多设备同时使用）时，自动关闭生物识别功能，需重新开通。

## 三、设备管理规则

### 3.1 设备绑定规则

| 规则项 | 规则说明 |
|--------|----------|
| 最大绑定数量 | 每位客户最多绑定5台设备 |
| 绑定条件 | 首次在新设备登录时需通过身份验证（登录密码+短信验证码+人脸识别） |
| 绑定流程 | 新设备登录→系统检测为新设备→触发身份验证→验证通过→自动绑定→推送通知至已绑定设备 |
| 超限处理 | 绑定设备数量已达上限时，需先解绑旧设备方可绑定新设备 |
| 解绑方式 | 在当前使用设备的手机银行中解绑其他设备，或至柜面解绑全部设备 |
| 解绑效果 | 解绑后该设备无法登录手机银行，下次登录需重新绑定 |

### 3.2 设备信任等级

| 信任等级 | 可用功能 | 适用场景 |
|----------|----------|----------|
| 完全信任 | 全部功能（转账、支付、理财等） | 本人常用设备 |
| 仅查询 | 仅可查询余额、明细等非交易功能 | 临时使用设备 |
| 限制交易 | 交易限额降低至单笔1000元/日累计5000元 | 安全性较低的设备 |

### 3.3 登录安全规则

| 规则项 | 规则说明 |
|--------|----------|
| 异地登录检测 | 当登录IP/GPS与常用位置不符时，增加人脸识别验证 |
| 新设备登录 | 新设备首次登录强制进行多因子身份验证 |
| 异常时段登录 | 0:00-6:00登录增加短信验证码验证 |
| 并发登录限制 | 同一账号同一时间仅允许1台设备在线，新设备登录将踢出旧设备 |
| 登录失败通知 | 连续3次登录失败后，推送安全预警通知至客户 |

## 四、个人信息修改规则

### 4.1 信息分类与修改权限

| 信息类别 | 包含字段 | 修改渠道 | 是否需要审核 | 生效时间 |
|----------|----------|----------|-------------|----------|
| 核心信息 | 姓名、证件号码、证件类型 | 柜面（必须） | 需主管授权 | 柜面即时生效 |
| 重要信息 | 预留手机号、国籍 | 柜面/远程银行视频面签 | 需身份核验 | T+1日生效 |
| 一般信息 | 联系地址、电子邮箱、紧急联系人 | 手机银行 | 无需审核 | 即时生效 |
| 职业信息 | 职业、工作单位、单位地址、单位电话 | 手机银行/柜面 | 无需审核 | 即时生效 |
| 财务信息 | 年收入、家庭收入、资产状况 | 手机银行/柜面 | 无需审核 | 即时生效 |
| 证件有效期 | 证件起始日期、到期日期 | 手机银行（上传照片）/柜面 | 系统审核（1-3个工作日） | 审核通过后生效 |

### 4.2 手机号修改特殊规则

1. **柜面办理**：修改预留手机号原则上需持本人有效身份证件至柜面办理。
2. **远程银行办理**：部分分行支持通过远程银行（视频面签）修改手机号，需完成人脸识别+视频通话身份核验。
3. **过渡期**：手机号修改后T+1日生效，生效前原手机号仍可用于短信验证。
4. **关联影响**：手机号修改将影响以下功能，客户需知悉：
   - 短信验证码接收
   - 动账通知短信
   - 登录手机号（如使用手机号登录）
   - 第三方快捷支付绑定的手机号
5. **安全等待期**：手机号修改后的24小时内，交易限额自动降低至单笔5000元/日累计2万元。

### 4.3 证件有效期更新规则

1. **到期提醒**：系统在证件到期前90天、60天、30天、7天分别推送更新提醒。
2. **过期限制**：证件过期后，账户交易功能受限（仅可入账不可出账），直至更新证件信息。
3. **更新方式**：手机银行上传新身份证正反面照片+人脸识别验证，系统自动OCR识别证件信息并提交审核。
4. **审核时效**：一般1-3个工作日完成审核，审核通过后自动恢复账户正常功能。

## 五、隐私管理规则

### 5.1 数据授权规则

| 授权类型 | 授权内容 | 默认状态 | 关闭影响 |
|----------|----------|----------|----------|
| 基础服务授权 | 姓名、证件号、手机号等基础信息用于账户服务 | 开户时已授权（必要授权） | 不可关闭（关闭即注销账户） |
| 位置信息授权 | GPS/基站定位用于风控和地区服务 | 需客户同意 | 部分功能受限（如附近网点、本地缴费项目） |
| 相机授权 | 拍照/扫码功能 | 使用时申请 | 扫码、拍照识别等功能不可用 |
| 通讯录授权 | 转账收款人快捷导入 | 使用时申请 | 需手动输入收款人信息 |
| 个性化推荐 | 基于客户画像推送产品推荐 | 默认开启（可关闭） | 关闭后仍可收到通用消息 |
| 第三方共享 | 与保险/基金等合作方共享客户信息 | 默认关闭 | 不影响核心银行服务 |

### 5.2 隐私合规要求

1. **最小必要原则**：仅收集提供服务所必需的最少个人信息。
2. **明示同意**：涉及个人信息收集使用的功能，均需在使用前获得客户明示同意。
3. **撤回权利**：客户可随时撤回非必要信息的授权，撤回后不影响核心银行服务。
4. **信息删除权**：客户注销账户后，个人信息将在法律法规规定的保存期限届满后予以删除或匿名化处理。
5. **信息可携带权**：客户有权获取其在我行存储的个人信息副本。

## 六、消息通知规则

### 6.1 通知分类与规则

| 通知类型 | 通知方式 | 默认状态 | 是否可关闭 | 免打扰规则 |
|----------|----------|----------|------------|------------|
| 动账通知 | APP推送+短信 | 开启 | 短信可关闭（需柜面），APP推送可关闭 | 不纳入免打扰 |
| 安全预警 | APP推送+短信 | 开启 | 不可关闭 | 不纳入免打扰 |
| 交易限额提醒 | APP推送 | 开启 | 可关闭 | 不纳入免打扰 |
| 登录提醒 | APP推送 | 开启 | 可关闭 | 不纳入免打扰 |
| 还款提醒 | APP推送+短信 | 开启 | 可关闭短信 | 不纳入免打扰 |
| 理财到期提醒 | APP推送 | 开启 | 可关闭 | 遵循免打扰时段 |
| 营销消息 | APP推送 | 开启 | 可关闭 | 遵循免打扰时段 |
| 活动推送 | APP推送 | 开启 | 可关闭 | 遵循免打扰时段 |

### 6.2 推送频率控制

| 控制维度 | 规则说明 |
|----------|----------|
| 每日推送上限 | 营销类消息每日不超过3条；安全类消息不设上限 |
| 免打扰时段 | 默认22:00-08:00不推送非安全类消息（客户可自定义） |
| 消息聚合 | 短时间内多条同类消息自动聚合为1条摘要通知 |
| 疲劳控制 | 连续7天未点击营销消息后，自动降低推送频率 |
