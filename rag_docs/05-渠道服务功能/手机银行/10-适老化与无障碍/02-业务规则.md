# 手机银行 — 适老化与无障碍业务规则

## 一、适老化改造标准

### 1.1 政策依据

本板块适老化改造依据以下国家政策和行业标准执行：

| 序号 | 政策/标准名称 | 发布机构 | 核心要求 |
|------|-------------|----------|----------|
| 1 | 《互联网应用适老化及无障碍改造专项行动方案》 | 工业和信息化部 | 2021年1月起实施，要求金融机构网站和移动互联网应用完成适老化改造 |
| 2 | 《移动互联网应用（APP）适老化通用设计规范》 | 工业和信息化部 | 对APP的界面设计、交互方式、安全验证等提出具体要求 |
| 3 | 《关于切实解决老年人运用智能技术困难的实施方案》 | 国务院办公厅 | 要求保留传统服务方式，推动智能化服务适应老年人需求 |
| 4 | 《银行保险机构信息科技无障碍建设指引》 | 银保监会 | 要求金融机构推进信息科技无障碍建设 |
| 5 | 《信息技术 互联网内容无障碍可访问性技术要求与测试方法》（GB/T 37668-2019） | 国家标准委 | 互联网内容无障碍技术标准 |

### 1.2 界面设计适老化标准

| 设计规范项 | 具体要求 | 验证方式 |
|-----------|----------|----------|
| 字体大小 | 正文文字不小于18dp（约14px），标题不小于22dp，金额数字不小于24dp | UI走查+自动化检测 |
| 字体类型 | 使用系统默认字体（支持随系统字体缩放），避免使用装饰性字体 | 视觉检查 |
| 行间距 | 行间距不小于字体大小的1.5倍 | UI走查 |
| 段落间距 | 段落间留白不小于字体大小的1倍 | UI走查 |
| 颜色对比度 | 文字与背景颜色对比度不低于4.5:1（大字体不低于3:1） | 对比度检测工具 |
| 图标尺寸 | 功能图标不小于48×48dp，且配有文字标签说明 | UI走查 |
| 触控区域 | 可点击元素的触控热区不小于48×48dp | 交互测试 |
| 页面层级 | 核心功能操作路径不超过3步，避免多级嵌套 | 路径测试 |
| 弹窗限制 | 大字版模式下禁止弹窗广告、浮层营销、自动播放视频 | 功能测试 |
| 超时设置 | 页面操作超时时间不低于标准版的2倍，且提供延时选项 | 功能测试 |

### 1.3 交互设计适老化标准

| 设计规范项 | 具体要求 | 说明 |
|-----------|----------|------|
| 手势简化 | 核心功能不依赖复杂手势（如双指缩放、三指滑动、长按等） | 所有功能可通过简单点击完成 |
| 操作确认 | 关键交易操作提供明确的二次确认，确认按钮突出显示 | 确认页面信息清晰、按钮醒目 |
| 错误处理 | 操作错误时提供清晰的中文错误说明和解决建议 | 避免使用技术术语和错误代码 |
| 进度指示 | 多步骤操作显示明确的步骤进度（如"第2步/共3步"） | 降低操作焦虑感 |
| 返回操作 | 每个页面均有明显的返回按钮，支持返回上一步 | 避免用户迷失在深层页面中 |
| 操作反馈 | 每次操作后给予明确的视觉+触觉+语音反馈 | 多感官确认操作结果 |

## 二、无障碍合规要求

### 2.1 WCAG标准遵循

我行手机银行无障碍改造参照WCAG（Web Content Accessibility Guidelines）2.1标准执行，目标达到AA级合规。

| WCAG原则 | 具体要求 | 实施措施 | 合规等级 |
|----------|----------|----------|----------|
| 可感知性 | 所有非文本内容提供替代文本 | 图片添加contentDescription，图标添加文字标签 | A级 |
| 可感知性 | 视频内容提供字幕 | 产品说明视频、操作指引视频添加中文字幕 | A级 |
| 可感知性 | 信息不依赖单一感官传达 | 颜色编码增加文字/图标辅助说明（如涨用红色+↑，跌用绿色+↓） | AA级 |
| 可操作性 | 所有功能可通过键盘操作 | 支持Tab键导航、Enter键确认、Esc键返回 | A级 |
| 可操作性 | 充足的交互时间 | 超时时间可延长，不强制限时操作 | A级 |
| 可操作性 | 不使用可能引发癫痫的闪烁内容 | 不使用每秒闪烁3次以上的内容 | A级 |
| 可理解性 | 页面操作可预测 | 相同功能在不同页面保持一致的操作方式 | A级 |
| 可理解性 | 输入辅助 | 表单提供清晰的标签和输入提示，错误时提供修正建议 | A级 |
| 健壮性 | 兼容辅助技术 | 完整兼容TalkBack/VoiceOver/Switch Control等辅助工具 | A级 |
| 健壮性 | 状态同步 | 页面内容变化时通知读屏软件（Live Region） | AA级 |

### 2.2 读屏软件兼容规则

| 兼容性要求 | 具体规则 | 测试方法 |
|-----------|----------|----------|
| 元素识别 | 所有可交互元素具有正确的role属性和contentDescription | TalkBack/VoiceOver逐项测试 |
| 焦点管理 | 焦点按视觉逻辑顺序移动，不跳步、不遗漏 | 线性导航测试 |
| 状态播报 | 按钮状态（选中/未选中/禁用/加载中等）正确播报 | 交互状态测试 |
| 动态内容 | 页面动态更新内容（如余额变动）实时播报 | 动账场景测试 |
| 表单填写 | 输入框标签、提示文字、错误信息正确播报 | 表单交互测试 |
| 弹窗处理 | 弹窗出现时焦点转移至弹窗，关闭后返回原位 | 弹窗场景测试 |
| 手势兼容 | 不拦截或冲突读屏软件的手势操作 | 全场景手势测试 |

### 2.3 色觉障碍友好规则

1. **涨跌颜色替代方案**：除红绿色彩外，增加文字标识（"涨"/"跌"）和箭头符号（↑/↓）。
2. **图表辅助**：图表中的数据系列除颜色区分外，增加不同的线型（实线/虚线/点线）或图案填充。
3. **状态标识**：成功/失败/警告等状态除颜色外，增加图标标识（勾号/叉号/叹号）。
4. **链接区分**：超链接除颜色变化外，增加下划线标识。

## 三、大字版功能管理规则

### 3.1 版本切换规则

| 规则项 | 规则说明 |
|--------|----------|
| 切换方式 | 首次登录时根据系统字体大小推荐切换；登录后可在"设置"中手动切换 |
| 切换保留 | 版本选择记忆在设备本地，下次登录自动进入上次选择的版本 |
| 功能差异 | 大字版仅包含常用核心功能（9个入口），完整版功能需切换回标准版 |
| 安全标准 | 大字版与标准版采用相同的安全认证标准，不降低安全性 |
| 数据同步 | 大字版与标准版共享账户数据，切换版本不影响账户状态和交易记录 |
| 升级策略 | APP更新后保持用户上次的版本选择，不强制切换 |

### 3.2 大字版交易规则

| 规则项 | 规则说明 |
|--------|----------|
| 交易限额 | 大字版交易限额与标准版一致（客户可在子女协助下自行设置更低限额） |
| 转账限额 | 默认单笔5万元/日累计20万元（与标准版一致） |
| 亲情限额 | 子女可为父母设置更低的交易限额（不低于单笔1000元/日累计5000元） |
| 免密支付 | 大字版默认开启小额免密支付（1000元以下免密），可在设置中关闭 |
| 交易确认 | 大额交易（5000元以上）强制语音播报交易信息供客户确认 |
| 风控规则 | 大字版交易纳入与标准版相同的风控监测体系 |

### 3.3 语音交互规则

| 规则项 | 规则说明 |
|--------|----------|
| 语音识别语言 | 默认普通话，支持切换至粤语、四川话等方言 |
| 语音指令范围 | 支持功能导航（"我要转账"）、交易操作（"给张三转1000元"）、信息查询（"查余额"） |
| 语音交易确认 | 语音发起交易后，仍需手动确认或语音确认（"确认转账"）方可提交 |
| 语音安全 | 语音输入的密码通过加密通道传输，不在本地存储语音数据 |
| 隐私保护 | 语音交互数据仅在设备端处理，不上传至云端（离线语音识别模式可选） |
| 环境适应 | 嘈杂环境下语音识别准确率可能降低，系统自动提示"建议使用手动输入" |

## 四、亲情账户管理规则

| 规则项 | 规则说明 |
|--------|----------|
| 绑定条件 | 子女和父母的手机银行账户均为正常状态 |
| 绑定方式 | 子女在自己的手机银行发起"亲情绑定"→父母在自己的手机银行确认同意 |
| 权限范围 | 子女仅可查看父母账户余额和交易明细，不可进行交易操作 |
| 交易通知 | 父母同意后，可将动账通知同步推送给子女 |
| 限额代管 | 子女可建议降低交易限额，需父母确认后生效 |
| 解绑规则 | 任一方可随时解除亲情绑定，解绑后对方立即失去查看权限 |
| 隐私保护 | 亲情账户功能遵循最小权限原则，子女不可修改父母的账户设置 |

## 五、服务时间与支持规则

| 规则项 | 规则说明 |
|--------|----------|
| 大字版服务时间 | 7×24小时可用 |
| 语音客服 | 7×24小时智能语音客服，人工客服时间8:00-20:00 |
| 专属客服热线 | 60岁以上客户拨打客服热线自动优先接入，无需按键导航 |
| 电话银行适老化 | 电话银行IVR提供"老年客户请按0"快速转人工服务 |
| 远程协助 | 子女远程协助功能7×24小时可用（需双方同时在线） |

## 六、合规审计与持续改进

| 审计项 | 频率 | 执行方式 |
|--------|------|----------|
| 适老化功能走查 | 每个版本发布前 | QA团队按设计规范逐项检查 |
| 无障碍兼容性测试 | 每个版本发布前 | 使用TalkBack/VoiceOver完成全流程测试 |
| 老年用户可用性测试 | 每半年一次 | 邀请60岁以上真实用户参与测试 |
| WCAG合规评估 | 每年一次 | 第三方机构进行无障碍合规评估 |
| 政策合规审查 | 政策更新时 | 合规部门审查新政策并推动改造 |
| 用户反馈收集 | 持续 | 通过客服渠道收集老年用户反馈并持续优化 |
