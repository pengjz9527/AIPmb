# 手机银行 — 客户服务与消息中心业务规则

## 一、客服服务时间规则

### 1.1 各渠道服务时间

| 服务渠道 | 服务时间 | 服务方式 | 备注 |
|----------|----------|----------|------|
| 智能客服机器人 | 7×24小时 | 文字对话 | 覆盖常见业务咨询、功能引导、自助办理 |
| 人工在线客服 | 工作日8:00-20:00，周末及节假日9:00-17:00 | 文字对话 | 智能客服无法解决时转人工 |
| 视频客服 | 工作日9:00-17:00 | 视频通话 | 需身份核验的复杂业务（如远程面签） |
| 电话客服（自助） | 7×24小时 | 电话IVR自助 | 余额查询、密码修改、挂失等自助业务 |
| 电话客服（人工） | 工作日8:00-20:00，周末及节假日9:00-17:00 | 电话通话 | 复杂业务咨询和投诉处理 |
| 工单服务 | 提交7×24小时，处理工作时间 | 异步处理 | 24小时内首次回复，72小时内处理完毕 |
| 老年客户专线 | 7×24小时 | 电话直达人工 | 60岁以上客户免IVR导航 |
| 外语客服 | 工作日9:00-17:00 | 电话/在线 | 英语为主，其他语言视分行能力而定 |

### 1.2 节假日服务安排

| 节假日类型 | 人工客服时间 | 智能客服 | 电话客服自助 |
|-----------|-------------|----------|-------------|

| 周末（周六/周日） | 9:00-17:00 | 7×24小时 | 7×24小时 |
| 系统维护日 | 视维护时段暂停 | 维护期间可能暂停 | 维护期间暂停 |

## 二、响应时效规则

### 2.1 在线客服响应时效

| 场景 | 响应时效目标 | 实际达标率目标 | 超时处理 |
|------|------------|--------------|----------|
| 智能客服首次回复 | ≤3秒 | ≥99% | 自动降级至常见问题列表 |
| 人工客服接入排队 | ≤60秒（VIP≤30秒） | ≥90% | 超过120秒提示预约回拨 |
| 人工客服消息回复 | ≤30秒 | ≥95% | 超过60秒自动提醒坐席 |
| 会话超时断开 | 客户3分钟无响应自动提醒，5分钟无响应自动结束 | — | 自动保存会话记录 |

### 2.2 电话客服响应时效

| 场景 | 响应时效目标 | 备注 |
|------|------------|------|
| IVR自助服务 | 按键响应≤1秒 | — |
| 人工坐席接听 | 80%的来电在20秒内接听 | 排队期间播放等待提示 |
| VIP客户接听 | 90%的来电在10秒内接听 | 系统自动识别VIP优先排队 |
| 老年客户接听 | 90%的来电在15秒内接听 | 自动识别60岁以上客户 |
| 预约回拨 | 预约后30分钟内回拨 | 客户可选择回拨时间段 |

### 2.3 工单处理时效

| 工单类型 | 首次回复时效 | 处理完成时效 | 超时处理 |
|----------|------------|------------|----------|
| 一般咨询 | 4小时内 | 24小时内 | 自动升级至主管 |
| 业务办理 | 8小时内 | 48小时内 | 自动升级并催办 |
| 投诉工单 | 2小时内 | 72小时内 | 12小时未回复自动升级至部门负责人 |
| 紧急问题 | 1小时内 | 4小时内 | 30分钟未回复自动电话回访 |
| 监管转办 | 1小时内 | 按监管要求时限 | 按监管要求严格时限管理 |

## 三、消息触达规则

### 3.1 推送频率控制

| 消息类型 | 推送频率限制 | 合并策略 | 说明 |
|----------|------------|----------|------|
| 动账通知 | 无限制（每笔均推送） | 不合并 | 安全类消息不限制 |
| 安全预警 | 无限制 | 不合并 | 安全类消息不限制 |
| 还款提醒 | 还款日前3天/1天/当天各1次 | 不合并 | 最多3次/还款周期 |
| 理财到期 | 到期前3天/1天/当天各1次 | 不合并 | 最多3次/产品 |
| 营销活动 | 每日≤3条 | 同类活动合并 | 避免营销骚扰 |
| 系统公告 | 不限 | 不合并 | 重要公告确保触达 |
| 工单回复 | 不限 | 不合并 | 确保处理进度通知 |

### 3.2 免打扰规则

| 规则项 | 规则说明 |
|--------|----------|
| 默认免打扰时段 | 22:00-08:00（客户可自定义调整） |
| 免打扰期间规则 | 营销类、活动类消息暂停推送，延至免打扰结束后推送 |
| 免打扰豁免 | 动账通知、安全预警、交易确认不受免打扰限制 |
| 客户自定义 | 客户可在"消息设置"中自定义免打扰时段 |
| 节假日策略 | 法定节假日不推送营销消息（客户可订阅） |

### 3.3 推送通道优先级

| 优先级 | 推送通道 | 适用场景 | 说明 |
|--------|----------|----------|------|
| 1 | APP内消息 | 所有消息类型 | 用户打开APP后查看 |
| 2 | 厂商推送通道 | APP关闭时推送 | 华为/小米/OPPO/VIVO/苹果APNs |
| 3 | 短信通知 | 重要消息（动账/安全/还款） | 确保关键消息触达 |
| 4 | 站内信 | 系统公告、长篇公告 | APP内消息中心永久保存 |

### 3.4 推送成功率保障

| 保障措施 | 说明 |
|----------|------|
| 多通道重试 | 厂商推送失败后自动切换至短信通道补发 |
| 推送监控 | 实时监控各通道推送成功率，低于95%触发告警 |
| 用户引导 | 引导用户开启APP通知权限，提升推送到达率 |
| 权限检测 | 登录时检测通知权限状态，未开启则提示引导 |

## 四、电子回单法律效力规则

### 4.1 电子回单法律合规要求

| 规则项 | 规则说明 |
|--------|----------|
| 法律依据 | 《中华人民共和国电子签名法》《中华人民共和国合同法》 |
| 电子签名 | 电子回单采用可靠的电子签名技术，符合法律对电子签名的要求 |
| 防篡改 | 电子回单采用数字签名+时间戳技术，确保内容完整性和不可篡改性 |
| 法律效力 | 电子回单与纸质回单具有同等法律效力，可作为诉讼证据使用 |
| 保存期限 | 电子回单在银行系统保存至少5年（自交易日起），客户可随时下载 |
| 验真服务 | 提供在线验真功能，第三方机构可通过回单编号和验证码在线验证回单真实性 |

### 4.2 电子回单使用规则

| 规则项 | 规则说明 |
|--------|----------|
| 下载方式 | 交易完成后即时下载，或在交易记录中历史下载 |
| 文件格式 | PDF格式，包含银行电子印章 |
| 回单内容 | 交易日期、交易金额、收款方/付款方信息、交易流水号、银行印章 |
| 打印效力 | 电子回单打印件与原件具有同等法律效力 |
| 批量下载 | 支持按时间范围批量下载交易明细和回单（打包为ZIP文件） |
| 邮件发送 | 支持将电子回单通过加密邮件发送至指定邮箱 |

## 五、智能客服机器人规则

### 5.1 智能客服能力边界

| 能力范围 | 具体说明 |
|----------|----------|
| 可处理场景 | 账户查询引导、产品介绍、费率咨询、操作指引、简单业务自助办理（如口头挂失、密码重置引导） |
| 不可处理场景 | 复杂投诉处理、账户纠纷调解、大额业务审批、法律诉讼相关咨询 |
| 转人工触发条件 | 客户主动要求转人工；智能客服无法理解问题；客户情绪激动（情绪识别）；涉及敏感业务 |
| 上下文保持 | 多轮对话保持上下文，同一会话内最多保持10轮对话上下文 |
| 知识更新 | 知识库每周更新，重大政策变更24小时内更新 |

### 5.2 智能客服转人工规则

| 触发条件 | 处理方式 |
|----------|----------|
| 客户发送"转人工"/"人工客服" | 立即进入人工排队 |
| 连续2次无法理解客户意图 | 主动询问是否需要转人工 |
| 连续3次回答未解决客户问题 | 主动推荐转人工 |
| 情绪识别为"愤怒"/"焦虑" | 优先转人工，缩短排队时间 |
| 涉及敏感话题（投诉/法律/监管） | 立即转人工 |
| 客户等待超过60秒未响应 | 自动提示转人工选项 |

## 六、工单管理规则

### 6.1 工单提交规则

| 规则项 | 规则说明 |
|--------|----------|
| 提交渠道 | 手机银行APP（主渠道）、电话客服、柜面、微信公众号 |
| 必填信息 | 问题类型、问题描述（≥20字）、联系方式 |
| 选填信息 | 附件（截图/文件）、期望处理时间、关联账户 |
| 附件限制 | 支持jpg/png/pdf格式，单个文件≤10MB，最多5个附件 |
| 重复提交 | 系统对重复问题自动合并工单，避免重复工单 |
| 匿名工单 | 不支持匿名提交，需实名认证后提交 |

### 6.2 工单评价规则

| 评价维度 | 评分标准 |
|----------|----------|
| 处理时效 | 非常满意（5分）/满意（4分）/一般（3分）/不满意（2分）/非常不满意（1分） |
| 处理结果 | 同上5级评分 |
| 服务态度 | 同上5级评分 |
| 不满意处理 | 1-2分评价自动触发回访，由主管跟进处理 |
| 评价时效 | 工单完成后7天内可评价，逾期自动默认"满意" |

## 七、常见问题库管理规则

| 规则项 | 规则说明 |
|--------|----------|
| 内容来源 | 客服高频问题、业务部门提供、监管政策变更、客户反馈 |
| 更新频率 | 每周至少更新1次，重大政策变更24小时内更新 |
| 审核流程 | 业务部门编写→合规部门审核→客服部门发布 |
| 版本管理 | 保留历史版本，支持版本对比和回滚 |
| 搜索优化 | 支持关键词搜索、模糊搜索、联想搜索 |
| 反馈机制 | 用户可对FAQ进行"有用/无用"评价，持续优化内容质量 |
