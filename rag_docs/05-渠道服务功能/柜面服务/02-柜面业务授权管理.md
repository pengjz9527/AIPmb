# 柜面业务授权管理

> 本文档系统阐述柜面授权体系的层级设置、授权事项清单、远程集中授权流程和授权风险控制要点，以授权矩阵表格形式直观展示各类业务的授权要求，为柜面操作风险管理提供参考。

---

## 一、概述

柜面业务授权管理是银行内部控制的重要组成部分，通过分级授权、双人复核等机制，确保高风险业务和大额交易经过适当层级的审核批准，有效防范操作风险和道德风险。

授权管理遵循以下原则：

- **分级授权**：根据业务风险程度和金额大小设定不同授权级别。
- **权责对等**：授权人对其授权范围内的业务承担审核责任。
- **不相容岗位分离**：经办人与复核人/授权人不得为同一人。
- **及时性**：授权不得延误客户正常业务办理。

---

## 二、授权级别体系

### （一）授权层级设置

| 授权级别 | 授权人 | 权限范围 | 系统实现 |
|---------|-------|---------|---------|
| **一级（柜员自办）** | 经办柜员 | 低风险、小额常规业务 | 系统自动处理，无需额外授权 |
| **二级（复核）** | 复核柜员 | 中等风险业务，需双人交叉复核 | 系统要求另一柜员刷卡/输密码确认 |
| **三级（主管授权）** | 营业主管/会计主管 | 高风险业务、大额交易、特殊业务 | 系统弹出授权窗口，主管刷卡/指纹/密码授权 |
| **四级（支行长授权）** | 支行行长/副行长 | 超高风险业务、超大额交易、例外事项 | 系统要求支行长级别用户授权 |
| **五级（上级行授权）** | 分行/总行相关部门 | 特殊例外事项、超权限业务 | 逐级上报审批 |

### （二）授权方式

| 授权方式 | 适用场景 | 技术实现 |
|---------|---------|---------|
| 现场授权 | 授权人在柜面现场 | 授权人刷员工卡+输入授权密码 |
| 远程授权 | 授权人不在现场（集中授权模式） | 通过远程授权系统传输业务影像，远程审核授权 |
| 指纹授权 | 高安全等级场景 | 授权人通过指纹仪进行生物认证 |
| 密码授权 | 常规授权场景 | 授权人输入6位授权密码 |

## 三、授权矩阵

### （一）现金业务授权矩阵

| 业务类型 | 金额范围 | 授权级别 | 备注 |
|---------|---------|---------|------|
| 现金存款 | 5万元以下 | 一级（柜员自办） | — |
| 现金存款 | 5万（含）-20万元 | 二级（复核） | 大额交易登记 |
| 现金存款 | 20万元（含）以上 | 三级（主管授权） | 大额交易报告 |
| 现金取款 | 5万元以下 | 一级（柜员自办） | — |
| 现金取款 | 5万（含）-20万元 | 二级（复核）+身份证件 | 大额取现登记 |
| 现金取款 | 20万（含）-50万元 | 三级（主管授权） | 需提前预约 |
| 现金取款 | 50万元（含）以上 | 四级（支行长授权） | 需提前1个工作日预约 |

### （二）转账业务授权矩阵

| 业务类型 | 金额范围 | 授权级别 | 备注 |
|---------|---------|---------|------|
| 行内转账 | 20万元以下 | 一级（柜员自办） | — |
| 行内转账 | 20万（含）-50万元 | 二级（复核） | — |
| 行内转账 | 50万元（含）以上 | 三级（主管授权） | — |
| 跨行转账 | 5万元以下 | 一级（柜员自办） | — |
| 跨行转账 | 5万（含）-50万元 | 二级（复核） | — |
| 跨行转账 | 50万元（含）以上 | 三级（主管授权） | — |
| 跨境汇款 | 等值1万美元以下 | 二级（复核） | 外汇合规审核 |
| 跨境汇款 | 等值1万美元（含）以上 | 三级（主管授权） | 外汇合规审核+用途审核 |

### （三）特殊业务授权矩阵

| 业务类型 | 授权级别 | 特殊要求 |
|---------|---------|---------|
| 账户挂失（书面） | 二级（复核） | 核实客户身份 |
| 账户解挂 | 三级（主管授权） | 核实客户身份+挂失凭证 |
| 密码重置 | 三级（主管授权） | 本人办理，人脸识别 |
| 账户冻结 | 三级（主管授权） | 需有法律文书依据 |
| 账户解冻 | 三级（主管授权） | 需有法律文书依据 |
| 冲正交易 | 三级（主管授权） | 当日冲正需说明原因 |
| 隔日冲正 | 四级（支行长授权） | 需书面说明+分行报备 |
| 错账调整 | 四级（支行长授权） | 需详细说明+相关凭证 |
| 客户信息修改（核心字段） | 三级（主管授权） | 姓名、证件号修改需严格审核 |
| 销户 | 二级（复核） | 确认无未了事项 |
| 重要空白凭证领用 | 三级（主管授权） | 双人领取、登记 |
| 印章使用 | 三级（主管授权） | 用印登记、专人保管 |

## 四、双人复核事项清单

以下业务须经双人交叉复核（经办柜员+复核柜员），确保操作准确：

| 序号 | 复核事项 | 复核内容 |
|-----|---------|---------|
| 1 | 大额现金收付 | 复核现金金额与系统录入金额是否一致 |
| 2 | 跨行转账 | 复核收款人信息、金额、开户行是否正确 |
| 3 | 票据处理 | 复核票据要素（日期、金额、签章）完整性 |
| 4 | 挂失与解挂 | 复核客户身份信息和挂失/解挂申请一致性 |
| 5 | 重要凭证出售 | 复核出售的凭证种类和数量 |
| 6 | 印章加盖 | 复核印章使用是否符合规定 |
| 7 | 外汇业务 | 复核汇率、金额、合规性 |
| 8 | 冲正交易 | 复核冲正原因和冲正金额 |
| 9 | 大额存款开户 | 复核存款金额、期限、利率 |
| 10 | 代客操作 | 复核操作内容与客户指令一致 |

## 五、远程集中授权

### （一）模式说明

远程集中授权是将原本由网点主管现场完成的授权工作，集中到分行或总行的远程授权中心统一处理。柜员通过系统将业务信息和影像资料传输至授权中心，由远程授权员审核并授权。

### （二）适用业务范围

| 业务类别 | 纳入远程授权的业务 |
|---------|-----------------|
| 开户类 | 个人开户、电子银行签约 |
| 大额交易 | 大额取现、大额转账 |
| 特殊业务 | 挂失、密码重置、冲正 |
| 理财销售 | 理财产品购买（双录审核） |
| 外汇业务 | 超额度结售汇 |

### （三）远程授权流程

```
柜员发起业务 → 系统判断需授权
        ↓
柜员拍摄并上传业务资料影像（身份证、凭条、现金等）
        ↓
系统将业务信息+影像推送至远程授权中心
        ↓
远程授权员审核：身份核实、凭证完整性、合规性
        ↓
   ┌────────┐
   审核通过  │  审核不通过
   ↓         ↓
远程授权    退回柜面
系统确认    柜员补充材料或拒绝办理
   ↓
柜面完成业务处理
```

### （四）远程授权审核要点

| 审核维度 | 审核内容 |
|---------|---------|
| 身份核实 | 身份证影像与现场人脸照片是否一致 |
| 凭证完整 | 业务凭条填写是否完整，签字是否清晰 |
| 金额准确 | 系统录入金额与凭条/现金金额是否一致 |
| 合规性 | 业务办理是否符合相关规定（限额、用途等） |
| 影像清晰 | 上传的影像资料是否清晰可辨 |

### （五）远程授权效率指标

| 指标 | 标准值 | 说明 |
|-----|-------|------|
| 平均授权时间 | ≤3分钟 | 从提交到授权完成 |
| 授权通过率 | ≥90% | 一次提交通过比例 |
| 退回率 | ≤10% | 影像不清或资料不全退回 |
| 服务时间 | 营业时间内全覆盖 | 部分行7×12小时 |

## 六、授权风险控制要点

### （一）权限管理

1. **最小权限原则**：每位员工仅拥有完成工作所需的最小系统权限。
2. **岗位分离**：经办与授权岗位严格分离，授权人不得为自己经办的业务授权。
3. **定期轮换**：关键岗位人员定期轮换，防止长期固定形成内控盲区。
4. **权限清理**：员工离职或调岗时即时清理系统权限。

### （二）操作规范

1. **本人操作**：授权必须本人完成，不得将授权密码/员工卡交由他人代操作。
2. **逐笔审核**：授权人应对每笔业务进行实质性审核，不得批量"盲目授权"。
3. **授权记录**：所有授权操作均留有系统日志，可追溯查询。
4. **异常监控**：对授权频率异常、非工作时段授权等行为进行系统预警。

### （三）监督检查

1. **事后监督**：分行事后监督中心对已授权业务进行抽查复核。
2. **内审稽核**：内部审计部门定期检查授权制度执行情况。
3. **录像回放**：调取监控录像核实授权人是否到场授权。
4. **问责机制**：对违规授权行为严格追责，造成损失的依法追究责任。

## 七、附注

- 授权金额标准和授权级别可能因各行风险管理策略不同而有所差异，本文档为通用参考。
- 远程集中授权的具体业务范围以各行授权中心发布的规定为准。
- 建议定期对授权矩阵进行评估和更新，适应业务发展和监管要求的变化。
- 授权管理应与柜面操作风险监测系统联动，实现风险预警的自动化。
