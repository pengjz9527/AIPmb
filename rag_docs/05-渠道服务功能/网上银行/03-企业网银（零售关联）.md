# 企业网银（零售关联）

> 本文档聚焦企业网银中与个人零售业务产生关联的功能模块，重点说明代发工资、代报销、企业年金等业务的流程与联动关系，为零售条线与企业条线的协同营销提供参考。

---

## 一、概述

企业网银是银行面向企业客户提供的电子银行服务渠道，主要服务于企业的日常资金管理、支付结算和财务运营。然而，企业网银中有多项功能直接涉及个人零售客户，构成银行对公与零售业务联动的重要纽带。理解这些关联功能，对于零售条线做好客户服务、提升综合贡献度具有重要意义。

**与零售关联的核心功能**：

| 功能模块 | 关联方向 | 零售关联度 |
|---------|---------|-----------|
| 代发工资 | 企业→个人账户 | ★★★★★ |
| 代报销 | 企业→个人账户 | ★★★★☆ |
| 企业年金查询 | 企业↔个人 | ★★★☆☆ |
| 集团账户管理 | 企业内部 | ★★☆☆☆ |
| 银企直连 | 企业系统→银行系统 | ★★★☆☆ |

## 二、代发工资

### （一）业务定义

代发工资是指企业客户通过企业网银将员工工资批量转入员工个人银行账户的服务。这是企业网银与零售业务关联最为紧密的功能，也是零售条线获取个人客户的重要批量获客渠道。

### （二）业务流程

```
企业HR/财务部门准备工资数据
        ↓
登录企业网银 → 进入"代发工资"模块
        ↓
上传代发文件（Excel/CSV格式，含员工账号、姓名、金额）
        ↓
系统校验：账号格式、户名校验、金额合计
        ↓
经办岗提交 → 复核岗审核（双人操作）
        ↓
选择发薪日期（即时发放 或 预约发放）
        ↓
系统批量执行转账
        ↓
员工个人账户收到工资入账
        ↓
生成代发结果报告（成功/失败明细）
```

### （三）代发文件规范

| 字段 | 要求 | 说明 |
|-----|------|------|
| 收款账号 | 必填，16-19位银行卡号 | 支持本行和他行账户 |
| 收款人姓名 | 必填，与账号一致 | 系统自动校验一致性 |
| 金额 | 必填，精确到分 | 单笔上限通常5万元 |
| 用途/摘要 | 选填 | 默认"工资"，可自定义 |
| 身份证号 | 选填 | 部分银行要求用于税务申报 |

### （四）数据安全要求

1. **文件加密**：代发文件上传过程采用HTTPS/TLS加密传输，文件内容AES加密。
2. **权限分离**：企业端设置经办岗和复核岗，经办岗负责制作代发文件，复核岗负责审核确认，不可由同一人兼任。
3. **操作留痕**：代发操作全程记录操作日志，包括操作人、操作时间、IP地址、文件哈希值。
4. **敏感信息脱敏**：代发结果报告中员工账号、金额等信息进行脱敏展示。
5. **数据留存**：代发数据在银行系统保留不少于5年，到期后按规定销毁。

### （五）与零售客户的联动关系

| 联动维度 | 具体说明 |
|---------|---------|
| **批量获客** | 企业代发工资员工自动成为银行零售客户，是零售AUM增长的重要来源 |
| **工资卡激活** | 零售条线需配合做好工资卡的激活、电子银行签约等后续服务 |
| **专属产品** | 可针对代发工资客户推出专属理财产品、消费贷优惠利率 |
| **交叉营销** | 基于工资流水数据，精准推荐信用卡、房贷、保险等产品 |
| **客户分层** | 根据代发金额进行客户分层，高管客户纳入私行/财富管理 |
| **数据价值** | 代发数据支撑零售客户的收入核实、信用评级和风控建模 |

### （六）常见问题处理

- **代发失败**：账号错误或户名不符导致单笔失败，资金退回企业账户，需企业修正后重新代发。
- **延迟到账**：跨行代发受人行支付系统影响，大额支付系统工作日8:30-17:00发起的交易实时到账，非工作时段走小额系统可能延迟。
- **退汇处理**：到账后收款账户异常（如已销户），资金退回企业，需在3个工作日内处理。

## 三、代报销

### （一）业务定义

企业通过网银将员工报销款批量转入员工个人账户，适用于差旅报销、费用报销等场景。

### （二）业务流程

1. 员工在企业内部系统提交报销申请并获批。
2. 企业财务通过企业网银"代报销"模块发起批量付款。
3. 操作流程与代发工资类似，需经办/复核双人操作。
4. 报销款入账至员工个人账户，摘要标注"报销"。

### （三）与零售的关联

- 报销入账的个人账户即为零售客户的借记卡账户。
- 频繁的报销入账有助于提升客户账户活跃度和日均存款。
- 可基于报销数据洞察客户消费偏好，为精准营销提供支撑。

## 四、企业年金查询

### （一）业务定义

企业年金是企业及其职工在依法参加基本养老保险的基础上，自主建立的补充养老保险制度。银行作为年金托管人或账户管理人，为企业提供年金管理服务。

### （二）企业网银端功能

- 企业HR可查看年金计划整体运行情况、缴费明细。
- 批量导入员工年金缴费数据。
- 查询年金投资收益报告。

### （三）个人端关联

- 员工可通过个人网银或手机银行"企业年金"模块查询个人账户余额、缴费记录、投资收益。
- 企业年金是零售条线了解客户长期资产的重要窗口，有助于做好全生命周期财富规划。

---

## 五、集团账户管理

### （一）业务定义

针对集团型企业客户，提供母子公司账户的归集、下拨、监控等资金管理服务。

### （二）与零售的关联

- 集团下属子公司的员工代发工资、代报销等业务可带来批量零售客户。
- 集团企业高管是私行/财富管理的重点目标客群。
- 零售条线可与公司条线联动，通过集团账户合作切入企业高管的个人金融服务。

---

## 六、银企直连

### （一）业务定义

银企直连是指银行系统与企业ERP/财务系统（如用友、金蝶等）通过专线或VPN直接对接，企业的财务操作直接在自有系统中完成，交易指令实时传送至银行处理。

### （二）与零售的关联

- 银企直连中涉及的批量代发、批量代报销等功能，最终落地到个人账户。
- 银企直连企业的员工是零售条线的优质目标客群，金融需求稳定、收入可核实。
- 可通过银企直连合作深化与企业的关系，拓展零售业务覆盖面。

## 七、零售条线协同营销建议

| 协同策略 | 具体措施 |
|---------|---------|
| 代发工资联动获客 | 公司条线拓展代发企业时，零售条线同步进驻企业做开卡、电子银行签约 |
| 工资卡专属权益 | 为代发工资客户配置专属理财、优惠费率贷款、信用卡快速审批等权益 |
| 企业主/高管营销 | 从企业网银客户中筛选企业主和高管，纳入私行/财富管理服务体系 |
| 数据驱动的精准营销 | 基于代发、报销等数据分析客户画像，开展定向产品推荐 |
| 企业团购活动 | 联合企业开展员工专属金融活动（如团购房贷、团办信用卡） |

---

## 八、附注

- 代发工资业务涉及个人所得税代扣代缴，银行仅提供资金划转服务，不承担税务申报责任。
- 企业网银功能的开通需企业到柜面签署《企业电子银行服务协议》。
- 各行企业网银的功能界面和操作路径可能存在差异，本文档以通用流程为准。
