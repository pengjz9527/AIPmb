# 在线客服功能体系

## 一、概述

在线客服是商业银行通过互联网渠道为客户提供实时文字交互服务的远程服务方式。随着移动互联网的普及和客户行为习惯的变化，在线客服已成为电话银行之外最重要的远程服务渠道。城商行和农商行通常部署多平台在线客服体系，覆盖官方网站、手机银行APP、微信公众号/小程序等触点，形成统一的客户服务入口。

当前在线客服体系普遍采用"智能机器人+人工坐席"双轨并行模式，通过智能路由实现客户问题的自动分流与高效处理，在控制运营成本的同时提升服务效率和客户体验。

## 二、多渠道在线客服部署

### 2.1 官网在线客服

- **入口位置**：银行官网首页悬浮窗、各业务页面右下角固定入口、帮助中心页面
- **技术实现**：基于Web的即时通讯组件，支持PC端浏览器直接打开，无需安装插件
- **身份识别**：已登录网银客户自动关联客户号，未登录客户以游客身份接入
- **功能特点**：支持网页内嵌操作引导，可跳转至具体业务页面

### 2.2 手机银行APP在线客服

- **入口位置**：APP首页"客服"图标、"我的"页面"客服中心"、各功能页面浮动按钮
- **技术实现**：原生或H5嵌入，与APP深度集成
- **身份识别**：自动关联手机银行登录态，获取客户号、设备信息、当前页面
- **功能特点**：
  - 可推送APP内业务办理卡片，客户一键跳转办理
  - 支持截屏自动发送至会话，方便问题描述
  - 与APP消息推送打通，可异步通知处理结果

### 2.3 微信公众号/小程序在线客服

- **入口位置**：公众号菜单栏"在线客服"、小程序内嵌客服组件
- **技术实现**：基于微信客服消息接口或第三方SDK
- **身份识别**：通过OpenID关联客户，已绑定银行卡的客户自动识别身份
- **功能特点**：
  - 利用微信模板消息实现异步通知
  - 支持富媒体消息（图文、卡券、小程序卡片）
  - 可在公众号文章中嵌入客服入口

## 三、核心功能模块

### 3.1 智能机器人自动应答

智能机器人是客户接入在线客服后的第一层服务：

- **欢迎语与菜单推送**：客户进入后自动推送欢迎语和常见问题快捷菜单
- **意图识别**：基于NLP技术识别客户问题意图，自动匹配答案
- **标准问答**：覆盖利率查询、产品介绍、网点查询、业务办理指南等高频问题
- **业务办理引导**：引导客户通过自助渠道（手机银行、网上银行）完成具体业务
- **转人工触发**：当机器人无法解答、客户主动要求、或识别到复杂/敏感问题时，自动转接人工坐席
- **7×24小时服务**：机器人全天候在线，确保非人工服务时段的客户需求得到基本响应

### 3.2 常见问题库（FAQ）

- **分类体系**：按业务条线分类（存款、贷款、理财、信用卡、电子银行、账户管理等）
- **标准问答对**：每个问题配置标准答案，支持富文本（含链接、图片、表格）
- **热门问题排序**：根据客户访问频次自动调整排序，高频问题优先展示
- **关联推荐**：回答一个问题后，自动推荐相关问题
- **版本管理**：FAQ内容支持版本控制，修改留痕，支持回滚
- **多渠道同步**：一次维护，官网/APP/微信等多渠道同步更新

### 3.3 人工在线服务

- **在线转接**：从机器人平滑转接至人工，会话上下文自动传递
- **客户信息预加载**：坐席接入时自动显示客户基本信息、当前会话摘要、机器人交互记录
- **快捷回复**：坐席可调用预设的标准回复模板，提高回复效率
- **协同办理**：坐席可推送业务办理链接、产品购买链接至客户端，引导客户在线完成
- **多语言支持**：部分银行提供英语等外语在线服务

### 3.4 会话管理

**（1）排队机制**
- 高峰期客户进入排队队列，显示排队位置和预估等待时间
- VIP客户优先排队，缩短等待时间
- 排队期间推送自助服务引导，尝试分流

**（2）转接机制**
- **技能组转接**：根据客户问题类型转接至对应技能组（如贷款组、信用卡组）
- **坐席间转接**：坐席可将复杂问题转接至专家坐席，附带问题摘要
- **跨渠道转接**：电话客服与在线客服之间可互相转接，会话记录随转
- **升级转接**：普通坐席无法解决的问题可升级至主管坐席

**（3）满意度评价**
- 会话结束后自动推送满意度评价（1-5星）
- 评价维度：响应速度、问题解决程度、服务态度
- 低分评价自动触发回访流程
- 评价结果纳入坐席绩效考核

### 3.5 工单创建与跟踪

- **工单创建**：对于无法在线即时解决的问题，坐席创建工单并分类编号
- **工单内容**：客户信息、问题描述、已采取措施、期望结果、紧急程度
- **工单流转**：根据问题类型分发至责任部门（如科技部、运营部、风险部）
- **进度查询**：客户可通过在线客服查询工单处理进度
- **超时预警**：工单处理超时限自动预警并升级
- **闭环管理**：处理完成后由客服回访确认，关闭工单

### 3.6 文件与图片传输

- **客户上传**：支持客户上传图片（如截图、证件照片）辅助问题描述
- **坐席发送**：支持发送产品说明文档、操作指南PDF、业务流程图等
- **安全控制**：文件上传大小限制（通常10MB以内）、格式白名单、病毒扫描
- **敏感信息检测**：自动识别并提示客户不要在会话中发送密码、完整卡号等敏感信息

### 3.7 多会话并发

- **坐席端**：单个坐席可同时处理3-5个在线会话（可配置上限）
- **客户端**：同一客户在不同渠道（如APP和微信）发起会话时，可选择合并或独立处理
- **会话保持**：客户暂时离开后会话保持一定时间（通常10-15分钟），超时自动关闭并保存记录
- **消息提醒**：坐席端新消息提醒、客户等待超时提醒

## 四、运营管理与数据分析

### 4.1 坐席管理

- 实时监控：监控坐席在线状态、会话处理量、平均响应时间
- 质检评分：按一定比例抽检会话记录，从专业性、规范性、态度等维度评分
- 绩效统计：按日/周/月统计坐席接待量、解决率、满意度、平均会话时长

### 4.2 数据报表

- **服务量报表**：按日/周/月统计总会话量、机器人解决量、人工处理量
- **渠道分布**：各渠道（官网/APP/微信）会话量占比及趋势
- **热点问题**：客户咨询高频问题TOP N，为产品优化提供输入
- **响应效率**：平均首次响应时间、平均会话处理时长、排队放弃率
- **解决率分析**：机器人解决率、人工一次解决率、工单完成率

### 4.3 知识库运营

- 定期更新FAQ内容，确保与产品变更、政策调整同步
- 收集机器人无法回答的问题，补充知识库
- 分析客户真实用语习惯，优化关键词匹配
- A/B测试不同答案版本的客户满意度，选取最优方案

## 五、安全与合规要求

- **会话加密**：全链路HTTPS加密传输，敏感数据加密存储
- **身份核验**：涉及账户敏感操作时，需通过短信验证码、交易密码等二次验证
- **信息脱敏**：会话中涉及的卡号、身份证号、手机号等自动脱敏显示
- **会话存档**：全部会话记录存档不少于5年，支持监管调阅
- **隐私保护**：遵守《个人信息保护法》，客户数据仅用于服务目的

## 六、发展趋势

在线客服正从"多渠道覆盖"向"全渠道融合"演进：客户在任一渠道发起的会话可无缝切换至其他渠道继续，上下文不丢失；智能机器人能力持续提升，逐步覆盖更多复杂业务场景；坐席辅助系统（Agent Assist）通过实时分析会话内容，自动向坐席推荐最佳答案和操作建议，显著提升服务效率和质量。
