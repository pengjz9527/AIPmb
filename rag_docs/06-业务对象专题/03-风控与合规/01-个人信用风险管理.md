# 个人信用风险管理

## 专题概述

个人信用风险管理是零售银行风险控制的核心领域。随着个人消费贷款、经营贷款、信用卡等零售信贷业务的快速发展，个人信用风险的管理能力直接决定了银行的资产质量和盈利能力。尤其在经济下行周期，个人违约率上升，信用风险管理的重要性更加凸显。

个人信用风险管理贯穿信贷业务的全生命周期，包括：**贷前信用评估**（评估借款人还款能力和意愿）、**贷中授信审批**（决定是否放款及放款条件）、**贷后风险监测**（跟踪借款人信用状况变化）、**不良资产处置**（回收违约贷款）。

对城商行/农商行而言，个人信用风险管理面临独特挑战：客户群体以中低收入为主，征信数据覆盖不完整，风控模型建设能力有限，贷后管理手段相对单一。近年来，大数据风控、智能审批等技术的应用，为中小银行提升信用风险管理水平提供了新的可能性。

## 详解内容

### 一、贷前信用评估

#### 信用评估数据源

| 数据源 | 内容 | 获取方式 | 权重参考 |
|--------|------|----------|----------|
| 人行征信报告 | 信贷历史、逾期记录、负债情况 | 征信查询授权 | 30%-40% |
| 银行内部数据 | 账户交易、存款、理财、历史还款 | 行内系统 | 20%-30% |
| 大数据征信 | 消费行为、社交数据、设备信息 | 第三方合作 | 10%-20% |
| 客户自报数据 | 收入、职业、房产、车辆 | 申请表填写 | 5%-10% |
| 政府数据 | 社保、公积金、税务、工商登记 | 数据共享/API | 10%-15% |
| 司法数据 | 被执行信息、失信记录、诉讼记录 | 司法数据平台 | 否决项 |

#### 人行征信报告解读要点

**核心关注指标**：

| 指标 | 正常范围 | 关注信号 | 否决信号 |
|------|----------|----------|----------|
| 当前逾期笔数 | 0 | 1-2笔 | ≥3笔 |
| 近2年累计逾期次数 | ≤3次 | 4-6次 | >6次 |
| 最长逾期天数 | 0 | 1-30天 | >90天 |
| 负债率（月还款/月收入） | ≤50% | 50%-70% | >70% |
| 近6月新增查询次数 | ≤3次 | 4-6次 | >6次（多头借贷） |
| 信用卡使用率 | ≤70% | 70%-90% | >90% |

#### 信用评分模型

**传统评分卡模型（Logistic Regression）**：

适用于城商行/农商行的经典评分卡模型：

```
信用评分 = 基础分 + 年龄得分 + 职业得分 + 收入得分 + 征信得分 + 行内数据得分

典型分段：
- 700分以上：优质客户，自动审批通过
- 600-699分：良好客户，标准审批
- 500-599分：一般客户，人工审批
- 500分以下：高风险客户，拒绝/附加条件
```

**大数据风控模型**：

| 模型类型 | 算法 | 适用场景 | 优势 |
|----------|------|----------|------|
| 逻辑回归 | LR | 传统信贷审批 | 可解释性强 |
| 梯度提升树 | XGBoost/LightGBM | 信用评分 | 准确率高 |
| 深度学习 | DNN/LSTM | 复杂行为分析 | 特征自动提取 |
| 图网络 | GNN | 团伙欺诈识别 | 关联关系挖掘 |
| 联邦学习 | FL | 跨机构数据合作 | 数据不出域 |

### 二、授信审批要点

#### 审批模式

| 模式 | 适用范围 | 时效 | 准确率 |
|------|----------|------|--------|
| 全自动审批 | 标准化产品、低风险客户 | 秒级 | >95% |
| 半自动审批 | 中等风险客户，系统推荐+人工复核 | 小时级 | >90% |
| 人工审批 | 复杂情况、大额贷款、特殊情况 | 天级 | >85% |

#### 额度与定价

**额度测算公式（消费贷）**：
```
授信额度 = MIN(收入×倍数, 需求金额, 产品上限) × 调节系数

其中：
- 收入×倍数：月收入 × 12 × (1-负债率) × 贷款年限系数
- 需求金额：客户申请金额
- 产品上限：产品设定的最高额度
- 调节系数：根据信用评分、客户等级调整（0.5-1.2）
```

**风险定价原则**：
- 基础利率 = 资金成本 + 运营成本 + 风险成本 + 目标利润
- 风险溢价 = f(信用评分, 产品类型, 担保方式, 贷款期限)
- 优质客户可享受利率优惠，高风险客户需支付更高风险溢价

#### 常见贷款类型审批要点

| 贷款类型 | 核心审批要素 | 关键风控点 |
|----------|-------------|-----------|
| 个人消费贷 | 收入证明、用途合理性 | 多头借贷、虚假用途 |
| 个人住房贷款 | 收入流水、首付款来源、房产评估 | 首付贷、收入造假 |
| 个人经营贷 | 经营流水、经营状况、行业风险 | 资金回流、空壳公司 |
| 信用卡 | 职业稳定性、征信记录 | 套现、恶意透支 |
| 农户贷款 | 种植/养殖规模、补贴收入 | 自然灾害、市场波动 |

### 三、贷后管理

#### 贷后监测维度

| 监测维度 | 监测指标 | 预警阈值 |
|----------|----------|----------|
| 还款行为 | 还款及时性、还款金额变化 | 逾期≥1天即预警 |
| 收入变化 | 代发工资变动、账户流水变化 | 月收入下降>30% |
| 负债变化 | 新增负债、征信查询频次 | 新增负债>月收入50% |
| 行为异常 | 提前还款、频繁申请、变更联系方式 | 突发变化 |
| 外部风险 | 行业下行、企业裁员、区域风险 | 外部事件触发 |

#### 贷后分级管理

| 客户等级 | 特征 | 管理策略 |
|----------|------|----------|
| 正常类 | 按时还款，无异常 | 常规监测 |
| 关注类 | 偶有逾期（≤30天），收入波动 | 加强监测，客户经理回访 |
| 预警类 | 连续逾期或重大风险信号 | 主动干预，催收前置 |
| 高危类 | 长期逾期或失联 | 重点催收，准备诉讼 |

### 四、不良资产处置

#### 催收体系

| 催收阶段 | 逾期天数 | 催收方式 | 执行主体 |
|----------|----------|----------|----------|
| 提醒期 | 1-7天 | 短信/APP推送/智能语音 | 系统自动 |
| 早期催收 | 8-30天 | 电话催收 | 行内催收团队 |
| 中期催收 | 31-90天 | 电话+上门催收+律师函 | 行内/外包 |
| 晚期催收 | 91-180天 | 委外催收 | 催收公司 |
| 诉讼催收 | >180天 | 司法诉讼/仲裁 | 法务部门 |

#### 催收合规要点

- 催收时间：不得在22:00-8:00之间催收
- 催收频次：合理频次，不得骚扰
- 催收对象：仅限借款人及其担保人
- 催收方式：不得使用暴力、威胁、侮辱等手段
- 信息保护：不得向第三方泄露借款人债务信息
- 催收记录：完整记录催收过程

#### 不良处置方式

| 处置方式 | 适用场景 | 回收率参考 |
|----------|----------|-----------|
| 自主催收 | 逾期早期、小额 | 30%-60% |
| 委外催收 | 逾期中期、分散 | 15%-40% |
| 诉讼执行 | 有资产、大额 | 40%-70% |
| 不良转让 | 批量处置 | 5%-30% |
| 核销 | 确无法回收 | 0%（账面处理） |
| 债转股/重组 | 有恢复可能 | 视情况 |


## 城商行/农商行实践

### 案例一：某城商行"大数据+传统"混合风控模式

该城商行在大数据风控能力有限的情况下，采用"线上+线下"混合模式：

**线上部分**（标准化消费贷）：
- 接入人行征信+第三方大数据
- 部署XGBoost评分模型，AUC达到0.82
- 70%以上的申请实现自动审批

**线下部分**（经营贷、大额贷款）：
- 客户经理实地走访，核实经营状况
- 收集"软信息"：社区口碑、邻居评价、家庭情况
- 通过"人熟地熟"优势弥补数据不足
- 建立"信贷委员会"集体审议大额贷款

### 案例二：某农商行"整村授信"信用风险管理

该农商行在农户贷款中采用"整村授信"模式：

- 以行政村为单位，对全村农户进行信用评定
- 成立"村级信用评定小组"（村干部+农户代表+客户经理）
- 根据评定结果给予差异化授信额度
- 利用农村"熟人社会"的声誉约束机制降低违约风险
- 建立"信用村""信用户"评定机制，守信村享受利率优惠

效果：整村授信区域不良率仅0.8%，远低于分散授信的2.5%。

### 案例三：某城商行"贷后智能预警系统"

该行部署贷后智能预警系统，实现风险的早发现、早干预：

- 每日自动扫描全量贷款的还款状态
- 结合客户账户流水、征信变化等数据生成预警信号
- 预警信号分级推送：自动提醒→客户经理→部门经理→分管行长
- 建立"逾期T+1回访"制度，逾期次日即电话回访了解原因
- 对短期困难客户（如临时失业）提供展期/调整还款计划方案

实施效果：M1逾期率下降25%，早期催收成功率提升至85%。

| 《商业银行信用卡业务监督管理办法》 | 信用卡风险管理要求 |

### 2. 授信管理要求

- 实行统一授信管理，防止过度授信
- 贷款审批须遵循"审贷分离"原则
- 大额个人贷款须双人审批
- 不得向不符合条件的关系人发放信用贷款

### 3. 催收管理要求

- 银行对委外催收机构负有管理责任
- 催收行为须合规，不得暴力催收、骚扰催收
- 催收外包须签订合规协议，明确行为规范
- 催收投诉应纳入银行投诉管理体系

### 4. 模型管理要求

- 银行使用的外部数据须合规获取
- 核心风控环节不得外包
- 风控模型须定期验证和更新
- 模型决策须具备可解释性

## 常见问题

**Q1：征信报告上的"查询次数多"真的会影响贷款审批吗？**
A：是的。征信查询次数过多（特别是"贷款审批"类查询）表明客户正在多处申请贷款，可能是资金紧张的信号。一般近6个月"贷款审批"查询超过6次会被视为多头借贷，增加拒贷概率。但"个人查询"和"贷后管理"类查询不影响。

**Q2：城商行/农商行的大数据风控能力如何提升？**
A：分三步走：第一步，充分利用行内数据（交易、存款、理财等）构建内部评分模型；第二步，接入合规第三方数据（运营商、社保、公积金等）丰富数据维度；第三步，引入机器学习算法提升模型精度。关键是数据质量和合规性，不要盲目追求"高大上"的模型。

**Q3：农户贷款的风险如何控制？**
A：农户贷款的核心风控手段包括：整村授信利用声誉约束、小额分散降低单笔风险、与农业保险结合分散自然风险、季节性还款安排匹配收入周期、担保/联保机制增强还款约束。

**Q4：如何平衡自动审批效率和风险控制？**
A：建立"自动审批+人工审核"的分层机制。低风险客户（评分高、金额小）自动审批；中风险客户系统推荐+人工复核；高风险客户纯人工审批。同时定期监控自动审批的逾期表现，动态调整自动审批阈值。

**Q5：不良贷款核销后还能继续催收吗？**
A：可以。核销是会计处理（将贷款从表内转入表外），不代表放弃债权。核销后银行仍有权继续催收，催收回收的资金计入营业外收入。核销贷款应"账销案存"，持续跟踪催收。
