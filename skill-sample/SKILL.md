# 财务健康分析器

从 coredatas/ 下的交易流水和信用卡账单中，分析用户财务健康状况，回答"收入增长能否覆盖消费增长"等前瞻性问题。

## 触发场景

- "分析我的财务状况"
- "我的收支情况怎么样？"
- "未来几年收入增长能不能覆盖消费增长？"
- "我存得下钱吗？"
- "我的消费趋势是什么？"
- "财务健康度分析"

## 分析流程

### Step 1: 运行分析脚本

```bash
python3 .qoder/skills/financial-health-analyzer/scripts/analyze_financials.py
```

脚本输出 JSON，包含：
- `monthly_trend` — 月度收支/结余时间序列
- `yearly_summary` — 年度汇总（收入/支出/结余/结余率）
- `growth_rates` — 年度增长率（收入增速 vs 支出增速）
- `income_sources` — Top 收入来源及占比
- `expense_categories` — Top 支出类别及占比
- `credit_card_trend` — 信用卡消费趋势
- `category_trend` — 消费分类年度趋势
- `anomalies` — 异常月份（奖金月、异常消费月）
- `account_summary` — 账户余额快照

### Step 3: 回答"收入能否覆盖消费增长"

核心判断逻辑：
1. 比较近3年**年化收入增速** vs **年化支出增速**
2. 若收入增速 > 支出增速 → 可覆盖，结余率改善
3. 若收入增速 < 支出增速 → 不可覆盖，结余率恶化
4. 识别**结构性消费**（刚性支出如房租/房贷/医疗）vs **弹性消费**（餐饮/购物/娱乐）
5. 结合**资产余额**（账户余额）判断缓冲能力

## 关键分析要点

### 收入端
- 识别**工资收入**占比（高频稳定对手方）
- 识别**奖金/年终**（异常高收入月份）
- 识别**公积金提取**、**理财产品赎回**等非持续收入

### 支出端
- 区分**转账汇款**（可能是房贷/房租/内部转账）与**消费支出**
- **信用卡还款**是负债转移非真实消费
- 关注**医疗健康**支出的趋势（是否有上升趋势）

### 预测框架
```
未来结余 = 收入 × (1 + 收入增速)^n - 支出 × (1 + 支出增速)^n
```
若支出增速 > 收入增速，差距会随时间**指数放大**，需尽早调整。
